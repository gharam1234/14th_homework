핵심요구사항) 공통컴포넌트는 다음의 규칙을 지켜 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.
            1) 공통컴포넌트 원본을 수정하지 말고, 아래의 props만 활용할 것.
                - Button 컴포넌트
                  * variant: "primary" | "secondary" | "tertiary"
                  * size: "small" | "medium" | "large"
                  * theme: "light" | "dark"
                  * className: 추가 스타일 적용 시 사용
                  * onClick: 클릭 이벤트 핸들러 (기능 구현 시)

            2) 적용될 공통컴포넌트 목록을 구현할 것.
                - 취소 버튼: <Button variant="secondary" theme="light" size="medium" />
                - 충전하기 버튼: <Button variant="primary" theme="light" size="medium" />

핵심요구사항) 프레젠테이션 컴포넌트 규칙을 준수할 것.
            1) Props로 데이터만 받아서 표시
            2) 상태 관리 없음 (useState, useReducer 금지)
            3) 이벤트 핸들러 없음 (onClick, onChange 등 금지)
            4) Side effect 훅 없음 (useEffect, useCallback 등 금지)
            5) UI 렌더링만 수행 (기능은 부모 컴포넌트에서 구현)

핵심요구사항) CSS 규칙을 준수할 것.
            1) CSS Modules만 사용 (.module.css)
            2) :global, :root, !important 사용 금지
            3) position: absolute 사용 금지 (Flexbox만 사용)
            4) gap 속성으로 간격 관리
            5) 있는 그대로만 구현 (불필요한 애니메이션 없음)
            6) globals.css 무분별한 변경 금지
