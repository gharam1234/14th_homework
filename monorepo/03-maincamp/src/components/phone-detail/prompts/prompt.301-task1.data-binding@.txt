구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - @01-common.mdc
            - @04-func.mdc

조건-파일경로) 구현/수정될 HOOK 파일: src/components/phone-detail/hooks/index.fetch.hook.ts
조건-파일경로) 수정될 컴포넌트 파일: src/components/phone-detail/index.tsx (header + basicInfo 영역)
조건-파일경로) 타입 정의 파일: src/components/phone-detail/types.ts (Phone, Seller, PhoneWithSeller, PhoneDetailProps 모두 싱크 맞출 것)
조건-파일경로) 테스트 파일: src/components/phone-detail/tests/basic-info.spec.ts (신규 작성)

==============================================

핵심요구사항) 더미 데이터를 제거하고 Supabase 실데이터를 바인딩할 것.

            1) 타입/데이터 스키마 정비
                - Phone/Seller/PhoneWithSeller 인터페이스에 header/basicInfo에서 사용하는 모든 필드를 포함시킬 것 (예: title, description, hashtags, images_urls, main_image_url, seller.nickname 등).
                - PhoneDetailProps에도 data/onShare/onBookmark 등 실제로 쓰이는 속성만 남기고 필수 여부를 명시할 것.
                - 타입 정비 후 index.tsx에서 타입 오류가 없어야 한다.

            2) Supabase 조회 로직 업데이트
                - useFetchPhoneDetail 훅에서 phones ↔ sellers 조인을 1회 호출로 처리하고, header/basicInfo에 필요한 필드를 모두 매핑한다.
                - phoneId가 없거나 데이터가 비어 있을 때의 에러 메시지를 한글로 정리한다.
                - DUMMY_PHONE_DATA는 스토리북 전용으로만 남기고, 페이지 진입 시에는 항상 Supabase 응답을 사용하도록 한다.

            3) Basic Info/헤더 UI 데이터 바인딩
                - 모델명: model_name → h1 타이틀
                - 판매자 표기: seller.nickname (없으면 "판매자 정보 없음")
                - 이미지: main_image_url + images_urls 배열을 useImageGallery에 그대로 전달
                - 가격: price, original_price, 할인율 계산 로직 유지 (original_price 없으면 할인율 hidden)
                - 해시태그: condition/battery_health를 기반으로 generateHashtags 결과를 data-testid="basic-info-hashtags"에 바인딩
                - 통화/숫자 포맷은 toLocaleString('ko-KR') 사용

            4) 헤더 액션 버튼 UX
                - 공유 버튼: onShare가 전달되지 않은 경우 navigator.clipboard.writeText(window.location.href)로 fallback 처리하고, 처리 결과를 토스트/alert로 노출
                - 위치 버튼: 클릭 시 data-testid="location-section"으로 scrollIntoView({ behavior: 'smooth' })
                - 북마크 버튼: useBookmarkHook을 헤더에서 호출하여 isBookmarked, bookmarkCount를 동기화하고, data-testid="bookmark-badge"에 상태 표기
                - 신고 버튼은 usePhoneDetailModalHook과 연동하여 모달 토글까지만 구현 (실제 신고 전송 X)

            5) 로딩/에러 상태 UI
                - PhoneDetailPage 내 로딩/에러/데이터 없음 UI는 유지하되, 메시지를 Supabase 조회 결과와 동기화
                - PhoneDetail 컴포넌트 내부에서도 data가 없으면 skeleton/placeholder 노출

            6) 테스트 (Playwright)
                - src/components/phone-detail/tests/basic-info.spec.ts 작성
                - page.waitForSelector('[data-testid="basic-info-section"]', { timeout: 500 }) 형태로 로드 완료 판단
                - 검증 항목: 모델명 텍스트, 할인율 계산 결과, 해시태그 생성, 북마크 버튼 토글, 위치 버튼 스크롤 동작(IntersectionObserver stub 또는 scroll 이벤트 spy)
                - jest / @testing-library/react 금지, networkidle 금지
                - 환경 변수 없이도 실행되도록 VALID_PHONE_ID를 상단 상수로 분리하고 README에 입력 방법 주석
