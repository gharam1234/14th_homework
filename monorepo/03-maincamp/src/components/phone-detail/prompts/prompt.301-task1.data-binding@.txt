==============================================

핵심요구사항) 다음의 기능을 playwright 테스트를 활용하여 TDD 기반으로 구현하고, 테스트에 통과할 때까지 반복할 것.

            1) 테스트 제외 라이브러리
                - jest
                - @testing-library/react

            2) 테스트 조건
                - timeout은 설정하지 않거나, 500ms 미만으로 설정할 것.
                - 페이지가 완전히 로드된 후 테스트할 것.
                    - 페이지 로드 식별 요구사항: 고정식별자 data-testid 대기 방법
                    - **중요금지사항** 페이지 로드 식별 금지사항: networkidle 대기 방법

            3) 테스트 API 조건
                3-1) 데이터
                    - 실제 API 데이터를 사용할 것.
                    - Mock 데이터를 사용하지 말 것.

                3-2) 성공 시나리오
                    - Supabase phones 테이블에서 폰 정보를 조회하여 바인딩

                3-3) 실패 시나리오
                    - API 호출 실패 또는 데이터 없음 처리

            4) 데이터 조건
                - 저장소: Supabase
                - 테이블: phones, sellers
                - 요청 파라미터: 폰 ID (URL 파라미터에서 가져옴)

핵심요구사항) 현재의 모든 구조를 그대로 유지하고, 다음의 Supabase 데이터를 실제로 바인딩할 것.

            1) 바인딩할 데이터 (phones 테이블)
                - model_name: 폰 모델명 (예: "iPhone 14 Pro")
                - condition: 단말기 상태 (예: "미개봉", "새것", "중고")
                - price: 판매가격
                - original_price: 정가
                - images_urls: 이미지 URL 배열

            2) 바인딩할 데이터 (sellers 테이블)
                - nickname: 판매자 닉네임

            3) 데이터 바인딩 상세내용
                - 모델명: {model_name}
                - 상태: {condition}
                - 판매가: {price}원
                - 정가: {original_price}원 (할인가 계산)
                - 할인율: ((original_price - price) / original_price * 100)%
                - 사양 해시태그: 조건별 해시태그 자동 생성
                    * battery_health >= 80: #배터리 좋음
                    * battery_health < 50: #배터리 나쁨
                    * condition === '미개봉': #미개봉
                    * condition === '새것': #새제품
                    * condition === '중고': #중고

            4) CSS 처리
                - 가격 색상: 정가보다 저렴한 경우 강조 색상
                - 해시태그: 배경색 적용
                - 반응형: 모바일에서 가격/상태 배치 조정

            5) 액션 버튼 라우팅 (각 버튼별로)
                - 신고 버튼: 모달 오픈 (추후 구현)
                - 공유 버튼: URL 복사/SNS 공유 (추후 구현)
                - 위치 버튼: 지도 섹션으로 스크롤 이동
                - 북마크 버튼: 북마크 토글 (useBookmark 훅 사용)

            6) 테스트
                - 모델명이 올바르게 표시되는지 확인
                - 할인율이 올바르게 계산되는지 확인
                - 해시태그가 조건에 맞게 생성되는지 확인
                - 액션 버튼이 클릭 가능한지 확인
