핵심요구사항) 다음의 기능을 playwright 테스트를 활용하여 TDD 기반으로 구현하고, 테스트에 통과할 때까지 반복할 것.

            1) 테스트 제외 라이브러리
                - jest
                - @testing-library/react

            2) 테스트 조건
                - timeout은 설정하지 않거나, 500ms 미만으로 설정할 것.
                - 페이지가 완전히 로드된 후 테스트할 것.
                    - 페이지 로드 식별 요구사항: 고정식별자 data-testid 대기 방법
                    - **중요금지사항** 페이지 로드 식별 금지사항: networkidle 대기 방법

            3) 테스트 API 조건
                3-1) 데이터
                    - 실제 API 데이터를 사용할 것.
                    - Mock 데이터를 사용하지 말 것.

                3-2) 성공 시나리오
                    - Supabase phones 테이블에서 스펙 정보를 조회하여 바인딩

                3-3) 실패 시나리오
                    - API 호출 실패 또는 스펙 정보 없음 처리

            4) 데이터 조건
                - 저장소: Supabase
                - 테이블: phones
                - 요청 파라미터: 폰 ID (URL 파라미터에서 가져옴)

핵심요구사항) 현재의 모든 구조를 그대로 유지하고, 다음의 Supabase 데이터를 실제로 바인딩할 것.

            1) 바인딩할 데이터 (phones 테이블)
                - manufacturer: 제조사 (예: "Apple", "Samsung")
                - model_name: 모델명 (예: "iPhone 14 Pro")
                - release_year: 출시년도 (예: 2022)
                - ram: 메모리 (예: "8GB", "12GB")
                - storage: 저장용량 (예: "256GB", "512GB")
                - battery_health: 배터리 건강도 (0-100)
                - color: 색상 (예: "Space Black", "Gold")
                - condition: 단말기 상태 (예: "미개봉", "새것", "중고")

            2) 데이터 바인딩 상세내용
                - 제조사: {manufacturer}
                - 모델: {model_name}
                - 출시년도: {release_year}년
                - RAM: {ram}
                - 저장용량: {storage}
                - 배터리 건강도: {battery_health}%
                - 색상: {color}
                - 단말기 상태: {condition}

            3) 데이터 포맷팅
                - 배터리 건강도: 프로그레스 바로 시각화
                    * 80 이상: 녹색
                    * 50-79: 노란색
                    * 50 미만: 빨간색
                - 출시년도: 현재 연도와의 차이 표시 (예: "2022년 (2년전)")

            4) CSS 처리
                - 그리드 레이아웃: 3열 (제조사/모델/출시년도), 3열 (RAM/저장용량/배터리), 2열 (색상/상태)
                - 배터리 건강도 프로그레스 바 색상 변화
                - 반응형: 모바일에서 1열 레이아웃으로 변경

            5) 테스트
                - 모든 스펙 정보가 올바르게 표시되는지 확인
                - 배터리 건강도 프로그레스 바가 올바르게 표시되는지 확인
                - 필드가 비어있을 경우 "정보 없음" 표시
