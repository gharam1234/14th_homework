핵심요구사항) 다음의 기능을 playwright 테스트를 활용하여 TDD 기반으로 구현하고, 테스트에 통과할 때까지 반복할 것.

            1) 테스트 제외 라이브러리
                - jest
                - @testing-library/react

            2) 테스트 조건
                - timeout은 설정하지 않거나, 500ms 미만으로 설정할 것.
                - 페이지가 완전히 로드된 후 테스트할 것.
                    - 페이지 로드 식별 요구사항: 고정식별자 data-testid 대기 방법
                    - **중요금지사항** 페이지 로드 식별 금지사항: networkidle 대기 방법

            3) 테스트 API 조건
                3-1) 데이터
                    - 실제 API 데이터를 사용할 것.
                    - Mock 데이터를 사용하지 말 것.

                3-2) 성공 시나리오
                    - Supabase phone_inquiries 테이블의 문의 목록을 조회하여 표시

                3-3) 실패 시나리오
                    - API 호출 실패 또는 문의 없음 처리

            4) 데이터 조건
                - 저장소: Supabase
                - 테이블: phone_inquiries, 사용자 정보 (조인)
                - 요청 파라미터: 폰 ID (URL 파라미터에서 가져옴)

핵심요구사항) 현재의 모든 구조를 그대로 유지하고, 다음의 Supabase 데이터를 실제로 바인딩할 것.

            1) 바인딩할 데이터 (phone_inquiries 테이블)
                - id: 문의 ID
                - buyer_id: 구매자 ID (사용자 정보와 조인하여 이름 조회)
                - question: 문의 내용
                - seller_reply: 판매자 답변
                - created_at: 문의 작성 시간
                - updated_at: 수정 시간

            2) 문의 목록 표시
                - 문의 없을 때 메시지: "등록된 문의가 없습니다."
                - 각 문의별로 다음 정보 표시:
                    * 구매자 닉네임
                    * 작성 시간 (상대시간: "1시간 전", "2일 전" 등)
                    * 문의 내용
                    * 판매자 답변 여부 (아이콘/배지)

            3) 판매자 답변 기능
                - 현재 로그인 사용자가 판매자인 경우만 표시
                - 답변 입력 필드 (textarea)
                - 답변 제출 버튼
                - 답변 제출 시 seller_reply 업데이트

            4) 구매자의 추가 문의(댓글) 기능
                - 구매자가 자신의 문의에만 댓글 추가 가능
                - 댓글 입력 필드 (수정으로 구현: question 추가 업데이트)
                - 댓글 제출 버튼

            5) 문의 삭제/수정 기능
                - 구매자만 자신의 문의 삭제/수정 가능
                - 삭제 확인 모달
                - 수정 모드: 기존 내용 미리 표시

            6) CSS 처리
                - 문의별 구분선
                - 답변 영역: 배경색으로 시각적 구분
                - 판매자/구매자 표시: 뱃지
                - 반응형: 모바일에서 문의 내용 축약 표시

            7) 테스트
                - 문의 목록이 올바르게 표시되는지 확인
                - 문의 없을 때 메시지가 표시되는지 확인
                - 판매자만 답변 기능이 보이는지 확인
                - 구매자만 삭제/수정 기능이 보이는지 확인
                - 답변 제출 시 Supabase에 업데이트되는지 확인
