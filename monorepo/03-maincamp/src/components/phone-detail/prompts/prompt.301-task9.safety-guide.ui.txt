구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - @01-common.mdc
            - @04-func.mdc

조건-파일경로) 수정될 컴포넌트 파일: src/components/phone-detail/index.tsx (새 guide 섹션)
조건-파일경로) 스타일 파일: src/components/phone-detail/styles.module.css
조건-파일경로) 테스트 파일: src/components/phone-detail/tests/safety-guide.spec.ts

==============================================

핵심요구사항) 안전거래 가이드 UI를 location 섹션 아래에 신규로 추가할 것.

            1) 섹션 구조
                - data-testid="safety-guide-section" 루트
                - 3개의 카드(직거래 팁 / 이런 거래는 피하세요 / 환불 정책)를 grid로 배치, 태블릿 2열, 모바일 1열
                - 각 카드 내부에 리스트 최소 3개 + 아이콘(SVG) + "자세히 보기" 링크

            2) 아코디언/인터랙션
                - 카드 제목 버튼을 누르면 내용이 토글되고 aria-expanded 값을 업데이트
                - transition: max-height 0.25s ease, overflow hidden
                - 호버 시 배경색 살짝 강조

            3) 텍스트 콘텐츠 (고정)
                - 직거래 팁: 밝은 장소, 정품 확인, 거래 내역 기록, 신분증 확인
                - 피해야 할 거래: 선입금, 시세 이하, 외부 플랫폼, 인증서 없음
                - 환불 정책: 7일 이내, 초기 상태, 배송료 구매자 부담, 손상 시 환불 불가

            4) 접근성
                - 리스트 항목은 `<li>` + data-testid="guide-item"
                - 아이콘은 aria-hidden 처리하고, 텍스트에는 명확한 대비 제공

            5) 테스트 (Playwright)
                - page.waitForSelector('[data-testid="safety-guide-section"]', { timeout: 500 })
                - 시나리오: 기본 렌더링, 각 섹션 토글 on/off, 반응형(브라우저 viewport 변경 후 열 수 계산), "자세히 보기" 링크 href 검증
                - jest / @testing-library/react 금지, networkidle 금지
