아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - @01-common.mdc
            - @02-wireframe.mdc
            - @03-ui.mdc

조건-파일경로) 구현될 TSX 파일경로: src/components/phone-detail/index.tsx
조건-파일경로) 구현될 CSS 파일경로: src/components/phone-detail/styles.module.css
조건-파일경로) 필요시 Hook 생성 경로: src/components/phone-detail/hooks/index.map.hook.ts

조건-외부라이브러리) Kakao Maps API를 사용하여 지도를 표시할 것.
                    - 공식 SDK: https://apis.map.kakao.com/web/documentation/
                    - API 키는 .env 파일에서 로드할 것 (NEXT_PUBLIC_KAKAO_MAP_API_KEY)
                    - 스크립트 로드는 useEffect에서 동적으로 처리할 것.

==============================================

핵심요구사항) 거래 희망 지역의 좌표(latitude, longitude)를 기반으로 Kakao Maps 지도를 렌더링할 것.
                1) 지도 표시 조건
                    - latitude와 longitude 데이터가 존재할 때만 지도 섹션 표시
                    - 데이터 없을 경우: 섹션 전체를 숨기거나 "등록된 정보가 없습니다" 메시지 표시

                2) 지도 초기화 로직
                    - 좌표를 Kakao Maps 지도 중심으로 설정
                    - 기본 줌 레벨: 15 (거리지도 기준 약 1km 범위)
                    - 마커를 좌표에 표시하고, 주소 정보 포함

                3) 마커 표시
                    - latitude/longitude에 마커 표시
                    - 마커 클릭 시 주소 정보 팝업 표시 (address, address_detail)
                    - data-testid 속성 추가: "phone-detail-map-marker"

핵심요구사항) 지도 UI 구조를 다음과 같이 구성할 것.
                1) 섹션 제목: "거래 희망 지역"
                2) 지도 컨테이너: Kakao Maps 렌더링 영역
                    - ID: "kakaoMap" (Kakao API 초기화 시 필요)
                    - data-testid: "phone-detail-map-container"
                3) 지도 아래 추가 정보: 주소 텍스트 표시
                    - data-testid: "phone-detail-map-address"

핵심요구사항) 다음의 스타일을 적용할 것.
                1) 지도 컨테이너
                    - 너비: 100% 또는 고정값
                    - 높이: 300px (조정 가능)
                    - 테두리: 1px solid #ddd 또는 global.css 토큰 활용
                    - 배경색: #f0f0f0
                    - 마진/패딩: 섹션 일관성 유지

                2) 주소 텍스트
                    - 폰트 크기: global.css 타이포그래피 토큰 활용
                    - 색상: global.css 색상 토큰 활용
                    - 여백: 지도 아래에 16px 정도 마진

핵심요구사항) 반응형 처리 및 에러 핸들링을 구현할 것.
                1) 에러 처리
                    - Kakao Maps API 로드 실패 시: 친절한 에러 메시지 표시
                    - 좌표 데이터 없을 때: 섹션 숨김 또는 메시지 표시

                2) 반응형
                    - 모바일 화면에서도 지도 크기 적절히 조정
                    - 줌 레벨 조정 (모바일에서는 더 좁은 범위 가능)

핵심요구사항) 다음 프롬프트(301 Routing)에서 사용할 수 있도록 Hook 구조를 설계할 것.
                - Hook 파일: src/components/phone-detail/hooks/index.map.hook.ts
                - Hook 역할: 좌표 유효성 검사, 마커 생성 로직 등을 분리
                - 다음 단계: 이 Hook을 활용하여 클릭 → Kakao Maps 링크 열기 기능 구현
