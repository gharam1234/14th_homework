아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - @01-common.mdc
            - @04-func.mdc

조건-파일경로) 구현/수정될 HOOK 파일: src/components/phone-detail/hooks/index.gallery.hook.ts
조건-파일경로) 수정될 컴포넌트 파일: src/components/phone-detail/index.tsx (imageGallery 영역)
조건-파일경로) 테스트 파일: src/components/phone-detail/tests/image-gallery.spec.ts

==============================================

핵심요구사항) Supabase에서 내려온 main_image_url, images_urls를 그대로 활용하도록 이미지 갤러리를 정비할 것.

            1) 데이터 연동
                - gallery 훅의 입력 값을 { mainImageUrl?: string; imageUrls?: string[] } 형태로 고정하고, PhoneDetail 컴포넌트에서는 phones.images_urls를 그대로 전달한다.
                - undefined/빈 배열 입력 시 PLACEHOLDER를 메인 이미지로 사용한다.

            2) 로딩/에러 상태 개선
                - onLoad/onError에서 setIsImageLoading/setImageError를 반드시 호출하고, 동일 이미지를 다시 선택했을 때 상태가 초기화되도록 resetGallery 유틸을 노출한다.
                - imageError 메시지는 data-testid="image-error"에만 표기하고 시각적 토스트는 노출하지 않는다.

            3) 썸네일/네비게이션 UX
                - 썸네일 버튼 data-testid는 `thumbnail-item-${index}`로 유지하되, keyboard 접근성을 위해 role="tab"/aria-selected를 부여한다.
                - 좌우 화살표 버튼은 focusable 해야 하며, 첫/마지막 인덱스에서 disabled 처리한다.
                - 모바일 스와이프 거리는 40px 이상일 때만 페이지를 넘기도록 상수화한다.

            4) 줌 모달
                - zoomLevel 최소/최대/step을 상수로 export하여 테스트에서 import 가능하게 한다.
                - ESC, overlay click, close 버튼이 모두 동일한 closeZoomModal을 호출하도록 통합한다.

            5) 테스트 (Playwright)
                - 기존 src/components/phone-detail/tests/image-gallery.spec.ts를 갱신한다.
                - page.waitForSelector('[data-testid="gallery-main-image"]', { timeout: 500 }) 로 로드 대기.
                - 검증 항목: placeholder 노출, 썸네일 클릭 시 src 변경, 좌우 화살표 disabled 상태, 스와이프 이동, 줌 모달 열림/닫힘, zoom in/out 버튼 활성화, 에러 메시지 표기.
                - jest / @testing-library/react 사용 금지, networkidle 대기 금지.
