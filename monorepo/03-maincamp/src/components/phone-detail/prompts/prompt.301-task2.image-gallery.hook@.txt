조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - @01-common.mdc
            - @04-func.mdc

==============================================

핵심요구사항) 이미지 갤러리 기능을 구현할 것. 다음의 기능들을 step-by-step으로 구현하고, 적용 결과를 체크리스트로 반환할 것.

            1) 메인 이미지 표시
                - Supabase phones 테이블의 main_image_url 표시
                - 이미지 로딩 상태 표시
                - 이미지 없을 경우 플레이스홀더 표시

            2) 썸네일 슬라이더 구현
                - images_urls 배열의 모든 이미지를 썸네일로 표시
                - 썸네일 클릭 시 메인 이미지 변경
                - 현재 선택된 썸네일 하이라이트 표시
                - 좌우 화살표로 슬라이더 스크롤 가능
                - 모바일에서는 스와이프 가능 (터치 이벤트)

            3) 이미지 줌 기능
                - 메인 이미지 클릭 시 확대 뷰 모달 오픈
                - 확대 뷰에서 마우스 휠 또는 핀치 제스처로 줌 가능
                - 확대 뷰 닫기 버튼 또는 ESC 키로 닫기

            4) 이미지 상태 관리
                - 현재 선택된 이미지 인덱스 상태
                - 로딩 상태
                - 에러 상태

            5) 성능 최적화
                - 이미지 lazy loading 적용
                - 불필요한 리렌더링 방지 (useCallback, useMemo 활용)

            6) 테스트 - Playwright TDD 기반 구현
                - 메인 이미지가 올바르게 표시되는지 확인
                - 썸네일 클릭 시 메인 이미지가 변경되는지 확인
                - 좌우 화살표 클릭 시 슬라이더가 스크롤되는지 확인
                - 이미지 없을 경우 플레이스홀더가 표시되는지 확인

==============================================

핵심요구사항) 다음의 기능을 playwright 테스트를 활용하여 TDD 기반으로 구현하고, 테스트에 통과할 때까지 반복할 것.

            1) 테스트 제외 라이브러리
                - jest
                - @testing-library/react

            2) 테스트 조건
                - timeout은 설정하지 않거나, 500ms 미만으로 설정할 것.
                - 페이지가 완전히 로드된 후 테스트할 것.
                    - 페이지 로드 식별 요구사항: 고정식별자 data-testid 대기 방법
                    - **중요금지사항** 페이지 로드 식별 금지사항: networkidle 대기 방법

            3) 테스트 API 조건
                3-1) 데이터
                    - 실제 API 데이터를 사용할 것.
                    - Mock 데이터를 사용하지 말 것.

                3-2) 성공 시나리오
                    - Supabase phones 테이블에서 폰 정보를 조회하여 이미지 URL 바인딩
                    - 메인 이미지(main_image_url) 표시 확인
                    - 썸네일 이미지들(images_urls 배열) 렌더링 확인
                    - 썸네일 선택 시 메인 이미지 동적 변경 확인
                    - 좌우 네비게이션 화살표 작동 확인

                3-3) 실패 시나리오
                    - API 호출 실패 시 에러 상태 처리 확인
                    - 데이터 없음(이미지 URL 없음) 시 플레이스홀더 표시 확인
                    - 이미지 로딩 실패 시 폴백 UI 표시 확인

            4) Playwright 테스트 파일 구성
                - 파일경로: src/components/phone-detail/tests/image-gallery.spec.ts
                - page.waitForSelector('[data-testid="gallery-main-image"]') 사용
                - page.click('[data-testid="thumbnail-item"]') 등의 요소 상호작용
                - page.isVisible() 또는 page.locator() 를 통한 검증
