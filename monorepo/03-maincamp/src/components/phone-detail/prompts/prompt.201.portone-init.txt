구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - @01-common.mdc
            - @04-func.mdc

조건-파일경로) 구현될 HOOK 파일경로: src/commons/hooks/usePortOneInit.hook.ts
조건-파일경로) 구현될 TEST 파일경로: src/commons/hooks/tests/usePortOneInit.spec.ts

==============================================

핵심요구사항) 포트원 SDK를 초기화하는 custom hook을 구현할 것.
                1) 환경변수에서 STORE_ID와 CHANNEL_KEY를 읽을 것.
                    - 환경변수 키: NEXT_PUBLIC_PORTONE_STORE_ID, NEXT_PUBLIC_PORTONE_CHANNEL_KEY
                    - 없으면 콘솔 에러 메시지 출력: "[에러타입] PORTONE_STORE_ID 또는 CHANNEL_KEY가 설정되지 않았습니다"

                2) 컴포넌트 마운트 시 PortOne SDK 초기화할 것.
                    - 초기화 함수: PortOne.setClientKey(clientKey)
                    - 클라이언트키 형식: "[STORE_ID].[CHANNEL_KEY]"
                    - SDK 로드 완료 후 상태 업데이트: isInitialized = true

                3) 반환값 구조를 정의할 것.
                    - isInitialized: boolean (SDK 초기화 완료 여부)
                    - storeId: string
                    - channelKey: string
                    - error: Error | null (초기화 실패 시 에러 객체)

핵심요구사항) 다음의 기능을 playwright 테스트를 활용하여 TDD 기반으로 구현하고, 테스트에 통과할 때까지 반복할 것.

                1) 테스트 제외 라이브러리
                    - jest
                    - @testing-library/react

                2) 테스트 조건
                    - timeout은 설정하지 않거나, 500ms 미만으로 설정할 것.

                3) 테스트 시나리오
                    3-1) 성공 시나리오: SDK 초기화 완료
                        - 환경변수가 모두 설정되어 있을 때
                        - hook 호출 시 isInitialized가 true로 변환되는지 확인

                    3-2) 실패 시나리오: 환경변수 누락
                        - NEXT_PUBLIC_PORTONE_STORE_ID가 없을 때
                        - NEXT_PUBLIC_PORTONE_CHANNEL_KEY가 없을 때
                        - error가 설정되고 isInitialized가 false인지 확인
                        - 콘솔 에러 메시지 출력 확인
