-태스크1. 페이지 진입·내비게이션
-요구사항 1. URL 파라미터로 중고폰 상세 데이터를 조회하고 초기 렌더링에 필요한 정보를 비동기 호출로 불러온다.
-요구사항 2. 상단 카테고리 탭·뒤로가기 버튼 클릭 시 지정 경로로 라우팅하며 SPA 상태를 유지한다.
-요구사항 3. 사용자 인증 정보를 확인해 판매 글 삭제, 즐겨찾기, 문의 작성 권한을 제어한다.

-태스크2. 상품 액션 처리
-요구사항 1. 삭제 아이콘 선택 시 확인 모달을 띄우고 승인되면 판매 글을 삭제한 뒤 목록으로 이동한다.
-요구사항 2. 링크 복사 아이콘 클릭 시 현재 상품 URL을 클립보드에 복사하고 성공/실패 토스트를 보여준다.
-요구사항 3. 위치 아이콘 클릭 시 거래 지역 지도를 열거나 외부 지도 서비스로 이동한다.
-요구사항 4. 즐겨찾기 토글 시 로그인 여부를 검증하고 관심상품 상태를 서버와 동기화한다.

-태스크3. 이미지 갤러리 동작
-요구사항 1. 썸네일 클릭·좌우 네비게이션으로 메인 기기 사진을 변경하고 선택 상태를 상태값으로 관리한다.
-요구사항 2. 이미지 로딩 실패 시 기본 중고폰 이미지를 노출하고 오류 로그를 기록한다.
-요구사항 3. 사진이 다섯 장 이상이면 스크롤 또는 “+N” 표시로 추가 이미지를 노출한다.

-태스크4. 구매 카드 로직
-요구사항 1. 판매가와 안내 문구를 API 응답으로 받아 통화 포맷에 맞춰 표시한다.
-요구사항 2. 구매 문의 버튼 클릭 시 로그인 여부, 판매 상태를 검증하고 채팅/결제 플로우로 연결한다.
-요구사항 3. 판매자 카드 클릭 시 판매자 프로필 혹은 다른 등록 기기 목록으로 이동한다.

-태스크5. 상세 설명·거래 위치 데이터
-요구사항 1. 상세 설명 텍스트에 기기 상태, 구성품, 교환 가능 여부 등의 정보를 포맷 그대로 렌더링한다.
-요구사항 2. 거래 희망 지역 좌표를 기반으로 지도 섹션을 렌더링하고 클릭 시 외부 지도나 길찾기를 연다.
-요구사항 3. 데이터가 없으면 섹션을 숨기거나 “등록된 정보가 없습니다” 메시지를 출력한다.

-태스크6. 문의 작성 폼
-요구사항 1. 입력 글자 수를 실시간 계산해 100자 초과 시 입력을 차단하고 경고 메시지를 표시한다.
-요구사항 2. 내용이 있어야만 문의 남기기 버튼을 활성화하고 제출 시 POST 요청 후 리스트를 새로고침 없이 갱신한다.
-요구사항 3. 제출 성공 시 입력값을 초기화하고 실패 시 에러 안내를 표시한다.

-태스크7. 문의 리스트 관리
-요구사항 1. 문의 목록을 최신순으로 받아오고 신규 등록 시 Optimistic Update 또는 재조회로 반영한다.
-요구사항 2. 문의 작성자와 현재 로그인 사용자를 비교해 본인에게만 수정/삭제 기능을 노출한다.
-요구사항 3. 판매자 답변이 존재하면 같은 카드 내에 연결해 보여준다.

-태스크8. 링크 미리보기 기능
-요구사항 1. 문의 본문에 URL이 포함되면 메타정보를 비동기 요청으로 받아 카드 형태로 노출한다.
-요구사항 2. 메타정보 요청이 실패하면 기본 URL만 유지하고 오류를 로깅한다.
-요구사항 3. 카드 클릭 시 새 탭으로 링크를 열고 클릭 이벤트를 추적한다.

-태스크9. 답글/문의 수정 플로우
-요구사항 1. 연필 아이콘 클릭 시 해당 문의·답글을 수정 모드로 전환하고 기존 내용을 입력창에 채운다.
-요구사항 2. 수정 완료 시 PATCH 요청을 보내고 성공/실패에 따라 UI를 갱신하거나 메시지를 표시한다.
-요구사항 3. 취소 아이콘 선택 시 수정 전 내용으로 즉시 복원한다.

-태스크10. 대댓글(답글) 작성
-요구사항 1. 답글 달기 클릭 시 부모 문의 ID를 가진 입력창을 열고 제출 시 해당 ID로 연결된 대댓글을 생성한다.
-요구사항 2. 대댓글도 100자 제한·글자 수 카운터·제출/취소 버튼을 동일하게 적용한다.
-요구사항 3. 서버 응답을 받아 부모 문의 아래에 스레드로 정렬하고 권한 확인 후 본인에게만 삭제/수정 기능을 제공한다.
