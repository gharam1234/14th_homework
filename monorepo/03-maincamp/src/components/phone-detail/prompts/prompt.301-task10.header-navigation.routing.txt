조건-파일경로) 수정될 컴포넌트 파일: src/components/phone-detail/index.tsx (헤더 영역)

==============================================

핵심요구사항) 헤더 버튼의 라우팅/공유/신고/북마크 동작을 훅으로 분리하고 Next.js 라우터 + 클립보드 API와 연동할 것.

            1) usePhoneDetailHeader 훅
                - 입력: { phoneId: string; sellerId: string; currentUserId?: string; shareUrl?: string }
                - 반환: onGoBack, onShare, onOpenReportModal, onScrollToLocation, bookmarkProps (useBookmarkHook 위임)
                - router.back()은 브라우저 히스토리가 없을 때 phones 목록 URL(`/phones`)로 fallback
                - 공유: navigator.share() 가능 여부 감지 후 → 가능 시 시스템 공유, 아니면 clipboard + 토스트

            2) 헤더 UI
                - 기존 버튼은 모두 onClick을 훅에서 받은 핸들러로 연결
                - 토스트는 공통 alert 대체 가능하되, 2초 후 자동 해제 로직(true/false state + setTimeout)
                - 더보기 드롭다운(optional)을 구현하되, Storybook에서도 동작하도록 router/mock 주입 가능하도록 props 추가

            3) 위치 이동
                - onScrollToLocation 호출 시 document.querySelector('[data-testid="location-section"]') 존재 여부 확인 후 smooth scroll

            4) 신고 모달
                - usePhoneDetailModalHook과 연동하여 payload(phoneId, phonePrice, userEmail)를 전달, 모달 열림까지 확인

            5) 테스트 (Playwright)
                - page.waitForSelector('[data-testid="header-section"]', { timeout: 500 })
                - 시나리오: 뒤로가기 → history.mock에 따라 back/fallback 경로 확인, 공유 버튼 → clipboard spy, 위치 버튼 → location 섹션이 viewport 안으로 들어오는지 확인, 북마크 버튼 토글
                - jest / @testing-library/react 금지, networkidle 금지
