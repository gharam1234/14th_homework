핵심요구사항) 다음의 기능을 playwright 테스트를 활용하여 TDD 기반으로 구현하고, 테스트에 통과할 때까지 반복할 것.

            1) 테스트 제외 라이브러리
                - jest
                - @testing-library/react

            2) 테스트 조건
                - timeout은 설정하지 않거나, 500ms 미만으로 설정할 것.
                - 페이지가 완전히 로드된 후 테스트할 것.
                    - 페이지 로드 식별 요구사항: 고정식별자 data-testid 대기 방법
                    - **중요금지사항** 페이지 로드 식별 금지사항: networkidle 대기 방법

            3) 테스트 라우팅 조건
                3-1) 네비게이션 검증
                    - 각 버튼 클릭 시 올바른 경로로 이동

                3-2) 상태 확인
                    - 버튼이 활성화/비활성화 상태 확인

핵심요구사항) 헤더 네비게이션 기능을 구현할 것.

            1) 뒤로가기 버튼
                - 위치: 헤더 좌측
                - 기능: 이전 페이지로 이동
                - 구현: useRouter().back()
                - CSS: cursor: pointer만 추가

            2) 공유 버튼
                - 위치: 헤더 우측
                - 기능: 현재 페이지 URL 공유
                - 공유 방식 (선택지):
                    * 클립보드 복사: URL 복사 후 "복사되었습니다" 토스트 메시지
                    * SNS 공유: 카카오톡, 페이스북, 트위터 등 (선택)
                    * 시스템 공유 API: navigator.share() 활용 (모바일)
                - 구현: 현재 페이지의 폰 ID를 URL에 포함하여 공유

            3) 더보기 메뉴 (선택)
                - 위치: 헤더 우측
                - 드롭다운 메뉴:
                    * 신고하기: 신고 모달 오픈
                    * 관심 상품 담기: 북마크 토글
                    * 공유하기: 공유 모달 오픈
                    * 채팅하기: 판매자와 채팅 페이지로 이동

            4) 상태 표시 (선택)
                - 북마크/관심 상태 아이콘
                - 채팅 읽지 않은 메시지 배지

            5) CSS 처리
                - 헤더 레이아웃: flexbox (뒤로가기 | 빈공간 | 공유/더보기)
                - 버튼 호버 효과: 배경색 변화
                - 토스트 메시지: 2초 후 자동 사라짐
                - 반응형: 모바일에서 버튼 크기 조정

            6) 테스트
                - 뒤로가기 버튼 클릭 시 이전 페이지로 이동하는지 확인
                - 공유 버튼 클릭 시 공유 메뉴가 열리는지 확인
                - URL이 올바르게 복사되는지 확인
                - 더보기 메뉴가 올바르게 작동하는지 확인
