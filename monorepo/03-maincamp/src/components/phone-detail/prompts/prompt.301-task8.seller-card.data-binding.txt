구현 결과를 체크리스트로 반환할 것.

            - @01-common.mdc
            - @04-func.mdc

==============================================

핵심요구사항) 다음의 기능을 playwright 테스트를 활용하여 TDD 기반으로 구현하고, 테스트에 통과할 때까지 반복할 것.

            1) 테스트 제외 라이브러리
                - jest
                - @testing-library/react

            2) 테스트 조건
                - timeout은 설정하지 않거나, 500ms 미만으로 설정할 것.
                - 페이지가 완전히 로드된 후 테스트할 것.
                    - 페이지 로드 식별 요구사항: 고정식별자 data-testid 대기 방법
                    - **중요금지사항** 페이지 로드 식별 금지사항: networkidle 대기 방법

            3) 테스트 API 조건
                3-1) 데이터
                    - 실제 API 데이터를 사용할 것.
                    - Mock 데이터를 사용하지 말 것.

                3-2) 성공 시나리오
                    - Supabase sellers 테이블에서 판매자 정보를 조회하여 바인딩

                3-3) 실패 시나리오
                    - API 호출 실패 또는 판매자 정보 없음 처리

            4) 데이터 조건
                - 저장소: Supabase
                - 테이블: sellers (phones 테이블의 seller_id 외래키를 통해 조인)
                - 요청 파라미터: 폰 ID (URL 파라미터에서 가져옴) → seller_id 추출

핵심요구사항) 현재의 모든 구조를 그대로 유지하고, 다음의 Supabase 데이터를 실제로 바인딩할 것.

            1) 바인딩할 데이터 (sellers 테이블)
                - profile_image: 판매자 프로필 이미지 URL
                - nickname: 판매자 닉네임
                - rating: 거래 평점 (0~5)
                - review_count: 평가 개수
                - temperature: 신뢰도 온도 (36.5~100)
                - phone_count: 판매 물품 개수

            2) 데이터 바인딩 상세내용
                - 프로필 이미지: {profile_image} (원형 아바타)
                - 닉네임: {nickname}
                - 거래 평점: {rating}점 ({review_count}개 평가)
                - 신뢰도: {temperature}°C (온도계 아이콘)
                - 판매 물품: {phone_count}개

            3) 신뢰도 온도 시각화
                - 온도에 따라 배경색 변화
                    * 36.5 - 50°C: 파란색 (낮음)
                    * 50 - 70°C: 노란색 (중간)
                    * 70 - 100°C: 빨간색 (높음)
                - 온도계 아이콘과 함께 표시

            4) 팔로우 기능
                - 현재 로그인한 사용자만 팔로우 가능
                - 팔로우 버튼 (토글)
                - 팔로우 상태 반영 (버튼 색상 변화)
                - 북마크 훅과 유사한 방식으로 구현

            5) 판매자 프로필 링크
                - 판매자 프로필 영역 클릭 시 판매자 상세 페이지로 이동
                - url.ts의 라우팅 경로 사용

            6) CSS 처리
                - 레이아웃: flexbox로 프로필(왼쪽) | 정보(중앙) | 팔로우버튼(오른쪽)
                - 프로필 이미지: 원형 (border-radius: 50%)
                - 신뢰도 온도: 프로그레스 바 또는 온도계 아이콘
                - 반응형: 모바일에서 세로 정렬

            7) 테스트
                - 판매자 정보가 올바르게 표시되는지 확인
                - 신뢰도 온도 색상이 올바르게 표시되는지 확인
                - 팔로우 버튼이 토글되는지 확인
                - 프로필 링크가 올바른 경로로 이동하는지 확인
