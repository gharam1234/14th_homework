핵심요구사항) 판매자 카드 UI를 추가하고 Supabase sellers 데이터를 그대로 바인딩할 것.

            1) useSellerCard 훅
                - 입력: { sellerId: string; currentUserId?: string }
                - Supabase에서 sellers 테이블 단건 조회
                - 팔로우 상태는 별도 followings 테이블이 없으므로 로컬 state + supabase rpc/fetch 대체 (임시) → 토글 시 POST / DELETE 호출 자리에 TODO 남김
                - temperature 구간별 배경색 클래스 반환

            2) Seller 카드 UI
                - data-testid="seller-card" 추가, 위치는 price 섹션 아래
                - 구성: 프로필 이미지(원형), 닉네임, 평점/리뷰수, 신뢰도 온도 게이지, 판매 물품 개수, 팔로우 버튼, 프로필 이동 CTA
                - 북마크 훅과 동일하게 로딩/에러 상태 표시
                - 프로필 영역 클릭 시 url.ts의 PHONE_DETAIL or SELLER_DETAIL 경로를 활용하여 router.push(`/sellers/${sellerId}`)

            3) 팔로우 버튼
                - 로그인하지 않은 경우 alert("로그인 후 이용 가능합니다")
                - 토글 시 optimistic update, 버튼 텍스트: "팔로우"/"팔로잉"

            4) 테스트 (Playwright)
                - page.waitForSelector('[data-testid="seller-card"]', { timeout: 500 })
                - 시나리오: 판매자 데이터 표시, 온도 색상 클래스, 팔로우 버튼 토글, 프로필 링크 이동(요청 취소 없이 navigation 확인)
                - jest / @testing-library/react 금지, networkidle 금지
