아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - @01-common.mdc
            - @04-func.mdc

조건-파일경로) 구현될 HOOK 파일: src/components/phone-detail/hooks/index.map.hook.ts (신규 생성)
조건-파일경로) 수정될 컴포넌트 파일: src/components/phone-detail/index.tsx (location-section)
조건-파일경로) 테스트 파일: src/components/phone-detail/tests/location-map.spec.ts (신규 작성)

==============================================

핵심요구사항) 거래 위치 정보를 지도/텍스트로 모두 표현할 것.

            1) usePhoneLocation 훅
                - 입력: { latitude?: number; longitude?: number; locationLabel?: string; tradeAvailable?: boolean; tradeTime?: string }
                - 출력: iframe src (Google 지도 embed URL), canRenderMap 여부, 거래 가능 뱃지 정보, 길찾기 링크, 주소 복사 핸들러
                - 훅 내부에서 window 객체 접근 시 안전하게 guard 할 것 (SSR 대비)

            2) 위치 섹션 UI
                - data-testid="location-section" 유지
                - 지도 영역: iframe + skeleton placeholder. 좌표 없으면 placeholder 카드와 "위치 정보가 없습니다." 문구 표시.
                - 거래 정보 박스: 거래 장소명, 거래 시간, 거래 가능 여부 뱃지(가능=green, 불가=gray), 길찾기 버튼(새 탭으로 `https://maps.google.com/?q=lat,lng`).
                - 주소 복사 버튼 클릭 시 navigator.clipboard.writeText(locationLabel) 수행 후 토스트/alert 노출.

            3) 성능/상호작용
                - iframe은 loading="lazy" 속성 사용.
                - 지도 스크롤 가로막지 않도록 pointer-events 제어(hover 시 활성화).
                - 거래 가능 여부 false일 때 지도 위에 반투명 오버레이 + 안내 문구 표시.

            4) 테스트 (Playwright)
                - page.waitForSelector('[data-testid="location-section"]', { timeout: 500 })
                - 시나리오: 좌표 있음/없음, 길찾기 링크 href 검증, 주소 복사 버튼 클릭 후 토스트 텍스트 확인, 거래 가능 여부에 따른 뱃지 클래스, 오버레이 존재 여부.
                - jest / @testing-library/react 금지, networkidle 금지.
