구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - @01-common.mdc
            - @04-func.mdc

조건-파일경로) 구현될 HOOK 파일경로: src/app/components/phone-detail/hooks/index.map.hook.ts
조건-파일경로) 기존 컴포넌트 파일: src/components/phone-detail/index.tsx (map 영역)

==============================================

핵심요구사항) 거래 위치 지도 기능을 구현할 것. 다음의 기능들을 step-by-step으로 구현하고, 적용 결과를 체크리스트로 반환할 것.

            1) 지도 라이브러리 선택
                - Naver Map API 또는 Kakao Map API 사용
                - 또는 Leaflet/OpenStreetMap 사용

            2) 지도 임베드 및 마커 표시
                - Supabase phones 테이블의 거래 위치 정보 (위도/경도) 조회
                - 지도에 마커 표시
                - 마커 클릭 시 거래 장소 정보 팝업 표시
                - 지도 줌 레벨 자동 조정

            3) 거래 가능 여부 표시
                - Supabase phones 테이블의 거래 가능 상태 확인
                - 거래 가능 여부를 텍스트/뱃지로 표시
                - 거래 불가능한 경우 지도에 오버레이 표시

            4) 위치 정보 표시
                - 거래 장소명 표시
                - 거래 시간 표시
                - 현재 거래 가능 여부 상태 아이콘 표시

            5) 상호작용 기능
                - 지도 드래그/확대축소 가능
                - 거래 장소명 클릭 시 주소 복사 기능
                - 길찾기 버튼 (Google Maps/Naver/Kakao 길찾기 연동)

            6) 성능 최적화
                - 지도 lazy loading
                - 불필요한 리렌더링 방지

            7) 테스트
                - 지도가 올바르게 로드되는지 확인
                - 마커가 올바른 위치에 표시되는지 확인
                - 거래 가능 여부가 올바르게 표시되는지 확인
                - 길찾기 버튼이 클릭 가능한지 확인
