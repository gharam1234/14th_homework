            - @04-func.mdc

조건-파일경로) 구현될 HOOK 파일경로: src/components/phone-detail/hooks/index.purchase.hook.ts (기존 파일에 추가)
조건-파일경로) 구현될 TEST 파일경로: src/components/phone-detail/tests/point-shortage-flow.spec.ts

==============================================

핵심요구사항) 포인트 부족 모달 상태를 hook에 추가할 것.
                - pointShortageModalOpen: boolean (포인트 부족 모달 열기/닫기)
                - openPointShortageModal: () => void (포인트 부족 모달 열기)
                - closePointShortageModal: () => void (포인트 부족 모달 닫기)

핵심요구사항) 포인트 부족 모달 흐름을 구현할 것.

                1) 모달 구조
                    - Modal 속성: variant="danger", actions="dual"
                    - 제목: "포인트 부족"
                    - 메시지: "구매에 필요한 포인트가 부족합니다. [필요한 포인트]P 필요 | [현재 포인트]P 보유"
                    - [취소] 버튼: 모달 닫기 (구매 안내 모달 상태 유지 또는 닫기)
                    - [충전] 버튼: 충전 모달로 전환

                2) 모달 상태 전환 로직
                    - 구매 안내 모달에서 [구매] 클릭 → FetchUserLoggedIn 호출
                    - 포인트 부족 확인 (point < price) → 포인트 부족 모달 자동 표시
                    - 포인트 부족 모달 [충전] → 구매 안내 모달 닫고 충전 모달 오픈

                3) 모달 일관성 규칙
                    - 모든 모달의 [취소]/[닫기] 버튼은 동일한 동작 구현
                    - 모달 간 전환 시 이전 모달 상태 정리

핵심요구사항) 충전 모달 상태를 hook에 추가할 것.
                - chargeModalOpen: boolean (충전 모달 열기/닫기)
                - openChargeModal: () => void (충전 모달 열기)
                - closeChargeModal: () => void (충전 모달 닫기)

핵심요구사항) 다음의 기능을 playwright 테스트를 활용하여 TDD 기반으로 구현하고, 테스트에 통과할 때까지 반복할 것.

                1) 테스트 제외 라이브러리
                    - jest
                    - @testing-library/react

                2) 테스트 조건
                    - timeout은 설정하지 않거나, 500ms 미만으로 설정할 것.
                    - 페이지가 완전히 로드된 후 테스트할 것.
                        - 페이지 로드 식별 요구사항: 고정식별자 data-testid 대기 방법

                3) 테스트 시나리오
                    3-1) 포인트 부족 모달 자동 표시
                        - 구매 [구매] 클릭 → FetchUserLoggedIn 호출
                        - 포인트 부족 조건 (point < price) 확인
                        - 포인트 부족 모달이 자동으로 표시되는지 확인

                    3-2) 포인트 부족 모달 [충전] 버튼
                        - [충전] 버튼 클릭 시 구매 안내 모달이 닫히는지 확인
                        - 충전 모달이 오픈되는지 확인

                    3-3) 포인트 부족 모달 [취소] 버튼
                        - [취소] 버튼 클릭 시 포인트 부족 모달이 닫히는지 확인

                    3-4) 모달 상태 정리
                        - 모달 간 전환 시 이전 모달 상태가 정확히 정리되는지 확인
