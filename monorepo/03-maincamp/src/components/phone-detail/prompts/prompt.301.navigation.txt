            1) 테스트 제외 라이브러리
                - jest
                - @testing-library/react

            2) 테스트 조건
                - timeout은 설정하지 않거나, 500ms 미만으로 설정할 것.
                - 페이지가 완전히 로드된 후 테스트할 것.
                    - 페이지 로드 식별 요구사항: 고정식별자 data-testid 대기 방법
                    - **중요금지사항** 페이지 로드 식별 금지사항: networkidle 대기 방법

핵심요구사항) url.ts의 라우팅 설정을 기반으로 다음의 네비게이션 기능을 구현할 것.

            1) 경로 참고
                - 파일: commons/constants/url.ts
                - 라우팅 유틸: getPath() 함수 사용

            2) 상단 카테고리 탭 클릭 시 라우팅
                - 요소: data-testid="category-tab-button"
                - 이동 경로: /phones (PHONES_LIST)
                - CSS: cursor: pointer만 추가할 것.

            3) 뒤로가기 버튼 클릭 시 라우팅
                - 요소: data-testid="back-button"
                - 이동 경로: /phones (PHONES_LIST)
                - 방식: router.push(getPath('PHONES_LIST'))
                - CSS: cursor: pointer만 추가할 것.

            4) SPA 상태 유지
                - router.push()를 사용하여 페이지 전체 새로고침하지 말 것.
                - 라우팅 시 SPA 상태(필터, 검색 등) 유지되도록 구현할 것.

            5) 테스트 시나리오
                - 카테고리 탭 클릭 시 /phones로 이동
                - 뒤로가기 버튼 클릭 시 /phones로 이동
                - URL이 정확히 변경되는지 검증 (await expect(page).toHaveURL('/phones'))
