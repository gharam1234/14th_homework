조건-파일경로) 구현될 TEST 파일경로: src/components/phone-detail/tests/purchase-modal-scenario.spec.ts

핵심요구사항) 구매 안내 모달의 3가지 모달 상태를 관리하는 custom hook을 구현할 것.
                - Modal 1: 구매 안내 모달 (구매하기 버튼 클릭 시)
                - Modal 2: 포인트 부족 모달 (사용자 포인트 부족 시)
                - Modal 3: 충전 모달 (포인트 충전)

핵심요구사항) 구매 안내 모달 시나리오를 구현할 것.

                1) 모달 오픈/클로즈 상태 관리
                    - purchaseModalOpen: boolean (구매 안내 모달 열기/닫기)
                    - openPurchaseGuideModal: () => void (구매 안내 모달 열기)
                    - closePurchaseGuideModal: () => void (구매 안내 모달 닫기)

                2) 구매 안내 모달 구조
                    - Modal 속성: variant="info", actions="dual"
                    - 제목: "[휴대폰 제목] 구매 확인"
                    - 메시지: "이 상품을 구매하시겠습니까?"
                    - [취소] 버튼 클릭 → /phones 페이지로 이동
                    - [구매] 버튼 클릭 → FetchUserLoggedIn 쿼리 호출

                3) FetchUserLoggedIn 쿼리 호출 및 분기 처리
                    - 쿼리 실행 후 사용자의 포인트(point) 값을 확인
                    - 조건 1: 포인트 >= 상품 가격 → "구현 예정" 안내 모달 표시
                    - 조건 2: 포인트 < 상품 가격 → 포인트 부족 모달로 전환

핵심요구사항) 다음의 기능을 playwright 테스트를 활용하여 TDD 기반으로 구현하고, 테스트에 통과할 때까지 반복할 것.

                1) 테스트 제외 라이브러리
                    - jest
                    - @testing-library/react

                2) 테스트 조건
                    - timeout은 설정하지 않거나, 500ms 미만으로 설정할 것.
                    - 페이지가 완전히 로드된 후 테스트할 것.
                        - 페이지 로드 식별 요구사항: 고정식별자 data-testid 대기 방법
                        - **중요금지사항** 페이지 로드 식별 금지사항: networkidle 대기 방법

                3) 테스트 API 조건
                    3-1) 실제 API 데이터 사용 (Mock 금지)
                    3-2) 성공 시나리오: FetchUserLoggedIn 쿼리 호출 및 포인트 확인
                    3-3) 실패 시나리오: API 호출 실패 처리

                4) 테스트 시나리오
                    4-1) 구매 안내 모달 오픈
                        - openPurchaseGuideModal() 호출 시 모달이 표시되는지 확인

                    4-2) [취소] 버튼 클릭 → /phones 이동
                        - 모달 [취소] 버튼 클릭 시 /phones 페이지로 라우팅되는지 확인

                    4-3) [구매] 버튼 클릭 시 FetchUserLoggedIn 호출
                        - FetchUserLoggedIn 쿼리가 호출되는지 확인

                    4-4) 포인트 충분할 때 (point >= price)
                        - "구현 예정" 메시지 모달이 표시되는지 확인

                    4-5) 포인트 부족할 때 (point < price)
                        - 포인트 부족 모달로 자동 전환되는지 확인
