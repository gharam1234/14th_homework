아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - @01-common.mdc
            - @02-wireframe.mdc
            - @03-ui.mdc

조건-파일경로) 구현될 TSX 파일경로: src/components/phone-detail/index.tsx
조건-파일경로) 구현될 CSS 파일경로: src/components/phone-detail/styles.module.css

==============================================

핵심요구사항) 기존 phone-detail 컴포넌트의 상세 설명 영역을 구현할 것.
                - 현재 DUMMY_PHONE_DATA의 description 필드 데이터를 포맷 그대로 렌더링할 것.
                - 줄바꿈, 들여쓰기, 불릿 포인트 등 원본 포맷을 그대로 유지할 것.

핵심요구사항) 상세 설명 섹션 구조를 다음과 같이 구성할 것.
                1) 섹션 제목: "상세 설명"
                2) 설명 본문: description 데이터를 <pre> 또는 <div>로 감싸서 공백/줄바꿈 유지
                3) 데이터 없을 경우: "등록된 정보가 없습니다" 메시지 표시

핵심요구사항) 다음의 스타일을 적용할 것.
                1) 섹션 컨테이너
                    - 배경색: 밝은 회색 또는 흰색 (기존 global.css 토큰 활용)
                    - 패딩: 컴포넌트 전체 일관된 여백 적용
                    - 테두리: 필요시 경계선 추가

                2) 텍스트 스타일
                    - 폰트 크기: global.css의 타이포그래피 토큰 활용
                    - 줄높이: 가독성 좋게 설정 (1.5 이상)
                    - 텍스트 색상: global.css의 색상 토큰 활용

                3) 공백/줄바꿈 유지
                    - white-space: pre-wrap 또는 white-space: pre-line 속성 적용
                    - 원본 포맷이 깨지지 않도록 처리
