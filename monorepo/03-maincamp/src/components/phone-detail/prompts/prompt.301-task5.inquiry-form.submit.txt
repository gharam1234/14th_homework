==============================================

핵심요구사항) PhonesInquiry 컴포넌트를 Supabase phone_inquiries 테이블과 연동하여 폼 제출이 가능하도록 만들 것.

            1) useInquiryForm 훅
                - 입력: { phoneId: string; sellerId: string; buyerId?: string }
                - 상태: value, errorMessage, isSubmitting, charsRemaining
                - 검증: 최소 5자, 최대 500자, 공백만 입력 불가 → 오류 메시지 한글로 노출
                - 제출: supabase.from('phone_inquiries').insert({ phone_id, seller_id, buyer_id, question, seller_reply: null })
                - 성공 시: "문의가 등록되었습니다." 토스트/alert, 입력값 초기화, onSuccess 콜백에서 목록 refetch 트리거

            2) PhonesInquiry 컴포넌트 연동
                - 기존 더미 데이터/로컬 상태 삭제
                - props에 { phoneId, sellerId, buyerId, onSubmitted } 추가
                - data-testid 유지: inquiry-textarea, submit-button 등
                - 제출 중에는 버튼 disabled + 로딩 스피너 아이콘 교체
                - 에러 메시지는 data-testid="inquiry-error"로 노출

            3) 문자 수 표시
                - "{현재 길이} / {최대 길이}" 형식 유지하되, 최대 길이는 훅 옵션으로 기본 500자

            4) 테스트 (Playwright)
                - page.waitForSelector('[data-testid="inquiry-input-section"]', { timeout: 500 })
                - 시나리오: 빈 입력 → 오류 메시지, 4자 입력 → 최소 길이 오류, 정상 입력 → Supabase insert spy 확인 또는 API 응답 모킹 없이 실제 키 사용, 성공 후 textarea 비우기, 에러 발생 시 재시도
                - jest / @testing-library/react 금지, networkidle 금지
