핵심요구사항) phone-detail 컴포넌트의 문의 목록 조회 및 작성 기능을 관리하는 커스텀 훅을 구현할 것.

1) 훅 상태 관리
   1-1) 다음의 상태를 관리할 것
        - inquiries: Inquiry[] (문의 목록)
        - inputContent: string (입력 중인 문의 내용)
        - characterCount: number (현재 문자 수)
        - isLoading: boolean (로딩 상태)
        - error: string | null (에러 메시지)

   1-2) Inquiry 타입 정의
        {
          id: string,
          userId: string,
          userName: string,
          content: string,
          createdAt: string,
          replies: Reply[]
        }

2) 초기 문의 목록 조회
   2-1) useEffect를 사용하여 컴포넌트 마운트 시 문의 목록을 조회할 것
        - GET /api/phones/{phoneId}/inquiries 호출
        - 응답: { inquiries: Inquiry[] }

   2-2) 조회 성공 시 inquiries 상태 업데이트

   2-3) 조회 실패 시 error에 에러 메시지 저장

3) 문의 입력 처리
   3-1) updateInputContent(content: string) 함수를 구현할 것
        - inputContent 상태 업데이트
        - 최대 100자까지만 허용 (초과 시 잘라내기)
        - characterCount 자동 계산

   3-2) 입력 검증
        - 공백만 입력할 수 없음
        - 최대 100자 제한

4) 문의 작성 함수
   4-1) createInquiry() 함수를 구현할 것
        - isLoading을 true로 설정
        - POST /api/phones/{phoneId}/inquiries 호출
        - 요청 본문: { content: string, userId: string }
        - 응답: { inquiry: Inquiry }

   4-2) 성공 시
        - 응답받은 inquiry를 inquiries 배열 맨 앞에 추가
        - inputContent를 빈 문자열로 초기화
        - characterCount를 0으로 초기화
        - 토스트 알림: "문의가 등록되었습니다."

   4-3) 실패 시
        - error에 에러 메시지 저장
        - 토스트 알림: "문의 등록에 실패했습니다. 다시 시도해주세요."

5) 문의 업데이트 함수
   5-1) updateInquiry(inquiryId: string, newContent: string) 함수를 구현할 것
        - PUT /api/inquiries/{inquiryId} 호출
        - 요청 본문: { content: string }

   5-2) 성공 시
        - inquiries 배열에서 해당 항목의 content 업데이트
        - 토스트 알림: "문의가 수정되었습니다."

   5-3) 실패 시
        - 이전 상태로 롤백
        - 토스트 알림: "수정에 실패했습니다."

6) 문의 삭제 함수
   6-1) deleteInquiry(inquiryId: string) 함수를 구현할 것
        - DELETE /api/inquiries/{inquiryId} 호출
        - 낙관적 업데이트: 먼저 UI에서 제거

   6-2) 성공 시
        - inquiries 배열에서 해당 항목 제거
        - 토스트 알림: "문의가 삭제되었습니다."

   6-3) 실패 시
        - inquiries에 항목 다시 추가 (롤백)
        - 토스트 알림: "삭제에 실패했습니다."

7) 훅 반환값
   훅은 다음 객체를 반환할 것:
   {
     inquiries: Inquiry[],
     inputContent: string,
     characterCount: number,
     isLoading: boolean,
     updateInputContent: (content: string) => void,
     createInquiry: () => Promise<void>,
     updateInquiry: (id: string, content: string) => Promise<void>,
     deleteInquiry: (id: string) => Promise<void>
   }

핵심요구사항) 테스트 시나리오
            1) 초기 목록 조회 테스트
               - GET 호출 확인
               - inquiries 상태 업데이트 확인

            2) 문의 작성 테스트
               - 입력 처리 확인
               - POST 호출 및 상태 업데이트 확인
               - 입력 필드 초기화 확인

            3) 문의 수정/삭제 테스트
               - PUT/DELETE 호출 확인
               - 상태 업데이트 및 롤백 확인

            4) 에러 처리 테스트
               - API 실패 시 error 상태 업데이트 확인
