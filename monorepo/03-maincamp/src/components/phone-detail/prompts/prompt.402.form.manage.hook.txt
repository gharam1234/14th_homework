아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - @01-common.mdc
            - @04-func.mdc

조건-파일경로) 구현될 HOOK 파일경로: src/components/phone-detail/hooks/index.form.manage.hook.ts
조건-파일경로) 구현될 TEST 파일경로: src/components/phone-detail/tests/index.form.manage.hook.spec.ts

==============================================

핵심요구사항) phone-detail 컴포넌트 내 답변 입력 폼의 통합 상태를 관리하는 커스텀 훅을 구현할 것.

1) 훅 상태 관리
   1-1) 다음의 상태를 관리할 것
        - currentInquiryId: string | null (현재 편집 중인 문의 ID)
        - formContent: string (폼에 입력된 내용)
        - isEditMode: boolean (편집 모드 여부)
        - editingReplyId: string | null (편집 중인 답변 ID)
        - characterCount: number (현재 문자 수)

2) 폼 초기화 메서드
   2-1) initializeCreateForm(inquiryId: string) 메서드를 구현할 것
        - currentInquiryId를 inquiryId로 설정
        - formContent를 빈 문자열로 초기화
        - isEditMode를 false로 설정
        - editingReplyId를 null로 설정
        - characterCount를 0으로 설정

   2-2) initializeEditForm(inquiryId: string, replyId: string, existingContent: string) 메서드를 구현할 것
        - currentInquiryId를 inquiryId로 설정
        - editingReplyId를 replyId로 설정
        - formContent를 existingContent로 설정
        - isEditMode를 true로 설정
        - characterCount를 existingContent.length로 계산

3) 입력 처리 메서드
   3-1) updateFormContent(newContent: string) 메서드를 구현할 것
        - formContent를 newContent로 업데이트
        - 최대 100자까지만 허용 (초과 시 잘라내기)
        - characterCount를 새로운 문자 수로 업데이트

   3-2) resetForm() 메서드를 구현할 것
        - currentInquiryId를 null로 초기화
        - formContent를 빈 문자열로 초기화
        - isEditMode를 false로 설정
        - editingReplyId를 null로 설정
        - characterCount를 0으로 설정

4) 검증 및 상태 확인 메서드
   4-1) canSubmit(): boolean 메서드를 구현할 것
        - 조건: formContent 길이 1 이상 100 이하
        - 공백만으로 이루어진 경우는 불가
        - 반환: 제출 가능 여부 (boolean)

   4-2) getCharacterCount(): number 메서드를 구현할 것
        - 반환: 현재 문자 수

   4-3) getRemainingCharacters(): number 메서드를 구현할 것
        - 반환: 남은 문자 수 (100 - characterCount)

   4-4) getCharacterCountDisplay(): string 메서드를 구현할 것
        - 반환: "{characterCount}/100" 형식의 문자열

   4-5) isEditMode(): boolean 메서드를 구현할 것
        - 반환: 편집 모드 여부

   4-6) isCreateMode(): boolean 메서드를 구현할 것
        - 반환: 생성 모드 여부 (isEditMode === false)

   4-7) getFormContent(): string 메서드를 구현할 것
        - 반환: 현재 입력된 폼 내용

   4-8) getEditingReplyId(): string | null 메서드를 구현할 것
        - 반환: 편집 중인 답변 ID

   4-9) getCurrentInquiryId(): string | null 메서드를 구현할 것
        - 반환: 현재 처리 중인 문의 ID

5) 유효성 검증
   5-1) 빈 문자열(공백만) 입력 검증
        - formContent.trim() 길이가 0이면 유효하지 않음

   5-2) 최대 100자 제한
        - 100자 초과 입력 시 자동 잘라내기

핵심요구사항) 훅 사용 예시
            const formManager = useFormManageHook();

            // 새 답변 작성 시작
            formManager.initializeCreateForm(inquiryId);

            // 입력 처리
            formManager.updateFormContent(e.target.value);

            // 화면 표시
            {formManager.getCharacterCountDisplay()} // "45/100"

            // 제출 가능 여부
            if (formManager.canSubmit()) {
              // API 호출
            }

            // 답변 수정 시작
            formManager.initializeEditForm(inquiryId, replyId, "기존 답변");

            // 폼 초기화
            formManager.resetForm();

핵심요구사항) 테스트 시나리오
            1) 폼 초기화 테스트
               - initializeCreateForm() 호출 후 상태 확인
               - initializeEditForm() 호출 후 상태 확인

            2) 입력 처리 테스트
               - updateFormContent() 호출 시 상태 업데이트 확인
               - 100자 초과 입력 시 자동 잘라내기 확인

            3) 검증 테스트
               - canSubmit(): 빈 문자열, 1자, 100자, 100자 초과 각각 확인
               - getCharacterCountDisplay(): 올바른 형식 확인

            4) 모드 확인 테스트
               - isEditMode(), isCreateMode() 확인

            5) 폼 초기화 테스트
               - resetForm() 호출 후 모든 상태 초기화 확인
