       - Supabase 저장 실패 시 에러 처리 검증

==============================================

핵심요구사항) 다음의 기능을 Supabase를 연동하여 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.
            1. 문의 등록 시나리오
                1-1) 다음의 데이터를 Supabase에 등록할 것
                        - 접속키: Supabase Client (환경변수 사용)
                        - 테이블명: phone_inquiries
                        - 데이터 (스키마 문서 기준)
                            - content: string (필수, 최대 100자, 빈값 불가)
                            - phone_id: UUID (필수, 현재 조회 중인 폰 ID)
                            - author_id: UUID (필수, 로그인한 사용자 ID, auth.users.id 참조)
                            - parent_id: UUID | null (대댓글용, 기본값 null)
                            - status: 'active' (기본값)
                            - is_answer: false (기본값, 판매자 답변 아님)

                1-2) 문의 등록 성공 이후 로직을 완성할 것.
                        - 알림메시지: "문의가 등록되었습니다."
                        - textarea 초기화
                        - 문의 목록 새로고침 (필요시)

                1-3) 실시간 글자 수 카운팅 기능
                        - 입력 중 실시간으로 글자 수 표시 (0/100 형태)
                        - 100자 초과 시 입력 제한 또는 경고 메시지

            2. 유효성 검증
                2-1) 필수 검증 규칙
                        - content: 빈값 불가, 최대 100자
                        - phone_id: 필수 (URL 파라미터에서 추출)
                        - author_id: 필수 (인증된 사용자만 작성 가능)

                2-2) 검증 실패 시
                        - 빈 내용: "문의 내용을 입력해주세요."
                        - 100자 초과: "문의 내용은 100자 이내로 작성해주세요."
                        - 인증 실패: "로그인이 필요합니다."

            3. 에러 처리
                3-1) Supabase 저장 실패 시
                        - 알림메시지: "문의 등록에 실패했습니다. 다시 시도해주세요."
                        - 입력한 내용 유지 (사용자 편의)
