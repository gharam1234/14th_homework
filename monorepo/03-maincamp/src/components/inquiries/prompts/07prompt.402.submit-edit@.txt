핵심요구사항) 인라인 수정 기능을 구현할 것.
                1) 수정 UI 토글
                    - 수정 버튼 클릭 시 내용 영역이 입력폼으로 변경
                    - textarea 또는 input[type="text"] 사용
                    - 최대 100자 제한
                    - 기존 content 값으로 초기화

                2) 수정 폼 액션
                    - 저장 버튼: 입력값 유효성 검증 후 Supabase 업데이트
                    - 취소 버튼: 수정 모드 종료, 입력값 폐기
                    - Enter 키: 저장 (선택사항)
                    - ESC 키: 취소 (선택사항)

                3) 유효성 검증
                    - 빈 값 불가 (trim 후 검증)
                    - 최소 1자, 최대 100자
                    - 검증 실패 시 에러 메시지 표시

                4) 로딩 상태
                    - Supabase 업데이트 중: 저장 버튼 비활성화, 로딩 스피너 표시
                    - 업데이트 완료: 수정 모드 종료, 리스트 업데이트

                5) 테스트 케이스
                    - 수정 버튼 클릭 시 입력폼 표시
                    - 수정된 내용이 Supabase에 반영되는지 확인
                    - 성공 메시지 표시 확인
                    - 취소 버튼으로 수정 모드 종료 확인
                    - 빈 값 제출 시 에러 메시지 확인
                    - 100자 초과 입력 차단 확인

핵심요구사항) 커스텀 훅 index.edit.hook.ts에 다음 로직을 구현할 것.
                1) 상태 관리
                    - isEditingId: 현재 수정 중인 답변 ID (null = 수정 모드 아님)
                    - editContent: 수정 입력값
                    - isLoading: Supabase 업데이트 중 상태
                    - error: 에러 메시지

                2) 함수
                    - startEdit(replyId, currentContent): 수정 모드 시작
                    - cancelEdit(): 수정 모드 취소
                    - updateContent(newContent): 입력값 업데이트
                    - submitEdit(replyId): Supabase 업데이트 실행
                        - 유효성 검증 후 Supabase 업데이트
                        - 성공 시 콜백 함수 실행
                        - 실패 시 에러 메시지 설정

                3) 반환값
                    - isEditingId, editContent, isLoading, error
                    - startEdit, cancelEdit, updateContent, submitEdit
