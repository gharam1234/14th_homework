핵심요구사항) 주소 검색 기능을 다음과 같이 구현할 것.
                1) 우편번호 검색 버튼
                    - "우편번호 검색" 버튼 클릭 시 모달 또는 다이얼로그 열기
                    - react-daum-postcode 라이브러리 사용 (또는 대체 라이브러리)

                2) 주소 검색 프로세스
                    - 검색 모달에서 주소 선택
                    - postalCode 필드에 우편번호 자동 입력
                    - detailedAddress 필드에 포커스 이동
                    - 사용자가 상세주소 직접 입력

                3) 좌표 자동 계산
                    - Daum 주소API에서 위도/경도 반환
                    - latitude, longitude 필드에 자동 입력

                4) 에러 처리
                    - 주소 검색 실패 시: alert "주소 검색에 실패했습니다."
                    - 좌표 계산 실패 시: alert "좌표 계산에 실패했습니다."

핵심요구사항) 로컬 상태 관리를 다음과 같이 구현할 것.
                1) isAddressModalOpen state (boolean)
                    - 주소 검색 모달 열림/닫음 상태 관리

                2) handlePostcodeSearch 함수 (기존 유지)
                    - 우편번호 검색 버튼 클릭 시 호출
                    - 주소 선택 로직 처리

                3) handleAddressSelect 함수 (신규)
                    - 주소 선택 완료 시 호출
                    - postalCode, detailedAddress (기본값) 설정
                    - latitude, longitude 설정
                    - 모달 닫기

                4) form 값 동기화
                    - setValue('postalCode', postalCode)
                    - setValue('latitude', latitude)
                    - setValue('longitude', longitude)

핵심요구사항) UI 요소를 다음과 같이 구현할 것.
                1) 주소 입력 섹션 (왼쪽)
                    - 우편번호 입력 (비활성화, readonly)
                    - 우편번호 검색 버튼
                    - 상세주소 입력 (활성화)
                    - 위도/경도 표시 (비활성화, readonly)

                2) 지도 섹션 (오른쪽)
                    - 제목: "거래 위치"
                    - 지도 컨테이너 (임시: 플레이스홀더)
                    - 실제 지도 구현 (향후)

                3) 스타일링
                    - 좌측과 우측 2단 레이아웃
                    - 좌측: 최대 400px, 우측: 유연한 너비
                    - 반응형: 모바일에서는 세로 배치

핵심요구사항) Daum Postcode API 연동을 다음과 같이 할 것.
                1) 라이브러리 설치
                    - npm install react-daum-postcode

                2) API 키 설정
                    - .env.local에 NEXT_PUBLIC_DAUM_API_KEY 설정

                3) 선택된 주소 정보
                    - zonecode (우편번호)
                    - address (기본 주소)
                    - addressType (주소 타입)
                    - latitude (위도) - Daum API에서 제공
                    - longitude (경도) - Daum API에서 제공

핵심요구사항) form 검증과 연동할 것.
                1) postalCode 필드
                    - 필수 필드
                    - 값이 없으면 "주소를 입력해 주세요." 에러 메시지

                2) detailedAddress 필드
                    - 필수 필드
                    - 값이 없으면 "상세주소를 입력해 주세요." 에러 메시지

                3) latitude, longitude 필드
                    - 자동 입력, 검증 불필요

핵심요구사항) Playwright 테스트를 TDD 기반으로 작성할 것.
                1) 테스트 파일경로: src/components/phone-new/tests/index.address.spec.ts
                2) 테스트 케이스
                    - 우편번호 검색 버튼 클릭: 모달 열림
                    - 주소 선택: postalCode 필드에 값 입력됨
                    - 주소 선택 후: detailedAddress 필드에 포커스 이동
                    - 지도 보임: 선택된 좌표 기반 지도 표시
                    - 모달 닫기: 모달 닫힘, 입력값 유지

                3) 테스트 조건
                    - timeout: 500ms 미만
                    - 페이지 로드 식별: data-testid 대기
