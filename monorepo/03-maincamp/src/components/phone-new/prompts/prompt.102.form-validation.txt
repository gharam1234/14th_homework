핵심요구사항) 폼 검증 UI를 다음과 같이 구현할 것. (react-hook-form 기반)
                1) 각 필드에 대한 에러 상태 표시
                    - 에러 발생 시 입력 필드에 errorBorder 스타일 적용
                    - 에러 메시지를 필드 아래에 빨간색으로 표시
                    - 에러 메시지 폰트: 12px, 색상: #d32f2f

                2) 필수 필드 표시
                    - 필드 라벨 옆에 빨간색 * 마크 표시
                    - 필수 필드: title, summary, description, price, postalCode, images
                    - 선택 필드: tags

                3) 실시간 validation 피드백
                    - 필드 blur 이후 에러 표시
                    - 에러 내용: 필드별 규칙에 따른 메시지

핵심요구사항) 폼 검증 규칙을 다음과 같이 적용할 것.
                1) title (기기명)
                    - 필수: "기기명을 입력해 주세요."
                    - 최소 길이: 3자 이상 "최소 3자 이상 입력해 주세요."

                2) summary (한줄 요약)
                    - 필수: "한줄 요약을 입력해 주세요."
                    - 최소 길이: 5자 이상 "최소 5자 이상 입력해 주세요."

                3) description (상품 설명)
                    - 필수: "상품 설명을 입력해 주세요."
                    - 최소 길이: 10자 이상 "최소 10자 이상 입력해 주세요."

                4) price (판매 가격)
                    - 필수: "판매 가격을 입력해 주세요."
                    - 숫자만 입력 가능: "숫자만 입력해 주세요."
                    - 최소값: 10000원 이상 "최소 10,000원 이상이어야 합니다."

                5) postalCode (우편번호)
                    - 필수: "주소를 입력해 주세요."
                    - 우편번호 검색을 통해 자동 입력

                6) images (사진 첨부)
                    - 필수: "사진을 최소 1개 이상 첨부해 주세요."
                    - 최대 2개: "최대 2개까지만 첨부 가능합니다."

핵심요구사항) 폼 제출 시 validation 수행 후 다음과 같이 처리할 것.
                1) 모든 validation 통과 시
                    - 제출 버튼 disabled 해제
                    - 클릭 가능 상태 (background-color: 파란색)

                2) validation 실패 시
                    - 제출 버튼 disabled 유지
                    - 클릭 불가 상태 (background-color: 회색)
                    - 첫 번째 에러 필드에 focus 이동

핵심요구사항) 다음의 UI 상태를 구현할 것.
                1) 정상 상태
                    - 입력 필드: 검은색 테두리, 배경: 흰색

                2) 에러 상태 (errors[fieldName] 존재)
                    - 입력 필드: 빨간색 테두리 (border-color: #d32f2f)
                    - 배경: 연한 빨간색 (background-color: #ffebee)
                    - 에러 메시지 표시

                3) 포커스 상태
                    - outline: 파란색
                    - box-shadow: 파란색 그림자

                4) 비활성화 상태 (disabled)
                    - 배경색: 연한 회색
                    - 텍스트색: 진회색
                    - cursor: not-allowed

핵심요구사항) Playwright 테스트를 TDD 기반으로 작성할 것.
                1) 테스트 파일경로: src/components/phone-new/tests/index.form.hook.spec.ts
                2) 테스트 케이스
                    - 필수 필드 빈 상태: 에러 메시지 표시 확인
                    - 필드별 규칙 위반: 각각의 에러 메시지 표시 확인
                    - 모든 필드 올바르게 입력: 에러 메시지 미표시, 제출 버튼 활성화 확인
                    - 이미지 없음: "사진을 최소 1개 이상 첨부해 주세요." 메시지 확인

                3) 테스트 조건
                    - timeout: 500ms 미만
                    - 페이지 로드 식별: data-testid 대기 (networkidle 금지)
