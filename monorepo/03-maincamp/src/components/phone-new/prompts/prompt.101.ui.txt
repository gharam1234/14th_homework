핵심요구사항) Figma 디자인을 기존 컴포넌트에 그대로 구현할 것.
                - 기존 코드의 form 구조, hooks, types, validation은 유지할 것.
                - usePhoneForm, usePhoneNewRouting 훅은 그대로 사용할 것.
                - 새로운 훅이나 state 추가는 하지 말 것.

핵심요구사항) 다음의 입력 필드들을 Figma 디자인 기준으로 구현할 것.
                1) 페이지 제목 (h1) - "중고폰 판매하기" / "중고폰 수정하기"
                2) 기기명 (title) - 필수, 텍스트 입력
                3) 한줄 요약 (summary) - 필수, 텍스트 입력
                4) 상품 설명 (description) - 필수, 에디터(B, I, U, S, 정렬, 목록, 링크, 이미지, 동영상)
                5) 판매 가격 (price) - 필수, 텍스트 입력
                6) 태그 입력 (tags) - 선택, 텍스트 입력
                7) 주소 섹션 (postalCode, detailedAddress, latitude, longitude)
                    - 우편번호: 비활성화된 상태
                    - 우편번호 검색 버튼: 클릭 가능
                    - 상세주소: 입력 가능
                    - 위도/경도: 표시만 (비활성화)
                8) 지도 섹션 - "거래 위치" 제목, 지도 컨테이너
                9) 사진 첨부 (images) - 필수, 최대 2개
                    - 미리보기 표시
                    - 삭제 버튼
                    - 업로드 버튼

핵심요구사항) 에러 처리를 다음과 같이 구현할 것.
                1) 각 필드에 에러 상태 CSS 클래스 적용 (style.inputError)
                2) 필수 필드에 빨간색 * 마크 표시
                3) 에러 메시지는 필드 아래에 작은 글씨로 표시
                4) form validation은 react-hook-form의 register된 필드에서 자동 수행

핵심요구사항) 폼 제출 버튼 상태를 다음과 같이 관리할 것.
                1) 제출 버튼 활성화 조건
                    - 모든 필수 필드가 입력되었을 때
                    - 이미지가 1개 이상일 때
                2) 활성화 상태: 파란색 배경, 클릭 가능
                3) 비활성화 상태: 회색 배경, 클릭 불가
                4) 로딩 상태: "처리 중..." 텍스트 표시, 클릭 불가

핵심요구사항) 버튼 섹션을 다음과 같이 구현할 것.
                1) 취소 버튼 (cancelButton)
                    - 흰색 배경, 검은색 테두리
                    - 클릭 시 폼 초기화 및 라우팅
                2) 제출 버튼 (submitButton)
                    - 신규: "등록하기"
                    - 수정: "수정하기"
                    - 로딩 중: "처리 중..."
