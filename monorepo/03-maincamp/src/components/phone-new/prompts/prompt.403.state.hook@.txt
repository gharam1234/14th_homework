핵심요구사항) 다음의 기능을 playwright 테스트를 활용하여 TDD 기반으로 구현하고, 테스트에 통과할 때까지 반복할 것.

1) 테스트 제외 라이브러리
   - jest
   - @testing-library/react

2) 테스트 조건
   - timeout은 설정하지 않거나, 500ms 미만으로 설정할 것.
   - 페이지가 완전히 로드된 후 테스트할 것.
       - 페이지 로드 식별 요구사항: 고정식별자 data-testid 대기 방법
       - **중요금지사항** 페이지 로드 식별 금지사항: networkidle 대기 방법

핵심요구사항) 상품 정보를 관리하는 상태 객체와 모드 관리 시스템을 구현할 것.

1) 상태 객체 타입 정의 (types/index.ts)
   - title: string (상품명, phones.title)
   - summary: string (한줄 요약)
   - description: string (상품 설명)
   - price: number (판매 가격)
   - tags: string[] (태그)
   - address: string (주소)
   - address_detail: string (상세주소)
   - zipcode: string (우편번호)
   - latitude: number (위도)
   - longitude: number (경도)
   - categories: string[] (카테고리 JSONB 매핑)
   - sale_state: 'available' | 'reserved' | 'sold'
   - mediaUrls: string[] (phone_media.url 목록)

2) 모드 타입 정의
   - 'draft': 초안 모드 (새로 작성)
   - 'completed': 완료 모드 (제출 완료)
   - 'edit': 수정 모드 (기존 데이터 수정)

3) 모드별 초기값 정의
   3-1) 초안 모드 (draft)
        - 모든 필드 빈 값으로 초기화
       - title: ''
        - summary: ''
        - description: ''
        - price: 0
        - tags: []
        - address: ''
       - address_detail: ''
       - zipcode: ''
       - latitude: 0
       - longitude: 0
       - categories: []
       - sale_state: 'available'
       - mediaUrls: []

   3-2) 완료 모드 (completed)
        - 제출된 데이터를 그대로 유지
        - 수정 불가 상태

   3-3) 수정 모드 (edit)
        - 기존 데이터를 불러와서 초기화
        - 모든 필드 수정 가능

4) 상태 관리 함수 구현
   4-1) updateField: 특정 필드 값 변경
        - 매개변수: (fieldName: keyof ProductState, value: any)
        - 기능: 지정된 필드의 값을 업데이트

   4-2) resetState: 전체 값 초기화
        - 매개변수: 없음
        - 기능: 현재 모드의 초기값으로 모든 필드 리셋

   4-3) switchMode: 모드 전환
        - 매개변수: (mode: 'draft' | 'completed' | 'edit', data?: ProductState)
        - 기능: 모드를 변경하고 해당 모드의 초기값으로 설정
        - edit 모드로 전환 시 data 매개변수로 기존 데이터 전달

   4-4) setInitialData: 외부 데이터로 초기화
        - 매개변수: (data: ProductState)
        - 기능: 전달받은 데이터로 상태 초기화 (수정 모드용)

5) 커스텀 훅 인터페이스 (usePhoneState)
   - 반환값:
       - state: ProductState (현재 상태)
       - mode: 'draft' | 'completed' | 'edit' (현재 모드)
       - updateField: (fieldName, value) => void
       - resetState: () => void
       - switchMode: (mode, data?) => void
       - setInitialData: (data) => void

6) 테스트 시나리오
   6-1) 초안 모드 초기화 테스트
        - 모든 필드가 빈 값으로 초기화되는지 검증

   6-2) 필드 업데이트 테스트
        - updateField로 각 필드 값 변경 시 정상 업데이트되는지 검증

   6-3) 리셋 테스트
        - resetState 호출 시 현재 모드의 초기값으로 돌아가는지 검증

   6-4) 모드 전환 테스트
        - draft → edit → completed 모드 전환 시 상태 변화 검증

   6-5) 수정 모드 데이터 로드 테스트
        - edit 모드로 전환하면서 기존 데이터가 정상 로드되는지 검증
