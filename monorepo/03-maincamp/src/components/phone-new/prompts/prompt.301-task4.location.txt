핵심요구사항) 판매자 위치 정보 섹션(Task 4)을 구현할 것. 이 섹션은 다음 3가지 필드를 포함한다:

1) 거래 장소 주소 입력 필드 (필수)
                - HTML: <input type="text"> 또는 주소 검색 API 연동
                - 라벨: "거래 장소 주소" (필수 표기 *)
                - 필드 너비: 100%
                - 높이: 40px
                - 필드명: tradeAddress
                - 플레이스홀더: "예) 서울시 강남구 테헤란로"
                - 에러메시지: 미입력 시 "주소를 입력해주세요"
                - data-testid: "phone-trade-address-input"

2) 지도 표시 영역 (필수, Kakao Map)
                - 라벨: "거래 장소" (지도 영역 위에 표시)
                - Kakao Map 컴포넌트 사용
                - 지도 높이: 300px
                - 지도 너비: 100%
                - 기능:
                    - 주소 입력 시 지도 자동 갱신 (주소 → 좌표 변환, Kakao Geocoding API)
                    - 마커 표시 (트레이드 위치)
                    - 기본 줌 레벨: 15
                - 주소 없으면: "주소를 먼저 입력해주세요" 메시지 표시 (지도 중앙에 텍스트)
                - data-testid: "phone-location-map"

3) 위치 상태 표시 (보조)
                - 주소 입력 전: "주소를 먼저 입력해주세요" (지도 영역 중앙, 회색 텍스트)
                - 주소 입력됨: 주소 텍스트 표시 (지도 아래, 작은 텍스트)
                - 주소 변환 중: 로딩 스피너 표시
                - 주소 변환 실패: "주소 조회에 실패했습니다" 에러 메시지
                - data-testid: "phone-location-status"

핵심요구사항) useLocation Hook을 구현할 것.
            1) 기능: 위치 관련 필드 상태 관리 및 지도 좌표 변환 (Supabase + 로컬 동시 관리)
            2) 상태
                - tradeAddress: string (Supabase phones.trade_address + 로컬스토리지)
                - latitude?: number (Supabase phones.latitude + 로컬스토리지)
                - longitude?: number (Supabase phones.longitude + 로컬스토리지)
                - isLoadingMap: boolean
                - mapError?: string
            3) 메서드
                - setTradeAddress(value: string)
                - searchAddressAndUpdateMap(address: string): Promise<void> (Kakao Geocoding API 호출)
                - setMapCoordinates(lat: number, lng: number)
                - reset(): void

핵심요구사항) 저장소 정책
            1) Supabase phones 테이블 필드 추가/활용: trade_address (text), latitude (float8), longitude (float8)
            2) Zustand persist 및 로컬스토리지: key = "phone-new-form-draft"에 동일 데이터 캐싱
            3) 폼 제출 시 Supabase에 저장 + 제출 성공 후 로컬 드래프트 삭제

핵심요구사항) Kakao Geocoding API 연동
            1) 사용 시점: tradeAddress 입력값 변경 시 (debounce 300ms)
            2) 보안 정책: 클라이언트에서 Kakao REST 키를 노출하지 말 것 → Next.js API Route 또는 Server Action(`/api/maps/geocode`)을 구현하여 서버에서 Kakao API 호출 후 좌표만 반환
            3) 서버 요청 규격
                - 클라이언트 → 사내 API: fetch('/api/maps/geocode', { method: 'POST', body: JSON.stringify({ query: address }) })
                - 서버 → Kakao: https://dapi.kakao.com/v2/local/search/address.json, Header Authorization: KakaoAK {KAKAO_REST_KEY}
            4) 응답 처리
                - 성공: 첫 번째 결과의 y(위도), x(경도) 추출 후 Zustand 상태 + Supabase 전송 준비
                - 실패: mapError 설정
            5) data-testid: "phone-location-geocoding"

핵심요구사항) 유효성 검증
            1) tradeAddress: 필수, 문자열, 최소 5자 이상
            2) latitude, longitude: 지도 표시 시 자동 계산
