핵심요구사항) 휴대폰 기본 정보 입력 섹션(Task 1)을 구현할 것. 이 섹션은 다음 3가지 필드를 포함한다:

1) 모델명 입력 필드 (필수)
                - HTML: <input> 태그 또는 <select> + combobox 패턴 사용
                - 자동완성 기능: Supabase phones 테이블에서 조회한 고유한 model_name 값 표시
                - 입력값 선택 시 해당 모델의 manufacturer, storage, ram, release_year 자동 채우기
                - 필드 너비: 100% (부모 컨테이너에 맞춤)
                - 높이: 40px
                - 라벨: "휴대폰 모델명" (필수 표기 *)
                - 에러메시지: 미입력 시 "모델명을 입력해주세요"
                - data-testid: "phone-model-input"

2) 기기 상태 선택 필드 (필수)
                - HTML: <select> 또는 Radio Button 형태
                - 옵션 (Supabase phones.condition 필드):
                    - 새것
                    - 거의새것
                    - 약간사용
                    - 많이사용
                - 기본값: "" (미선택)
                - 라벨: "기기 상태" (필수 표기 *)
                - 에러메시지: 미선택 시 "기기 상태를 선택해주세요"
                - data-testid: "phone-condition-select"

3) 색상 선택 필드 (필수)
                - HTML: <select> 또는 Color Input + Select 조합
                - 값: phones.color 필드
                - 기본값: "" (미선택)
                - 라벨: "색상" (필수 표기 *)
                - 에러메시지: 미선택 시 "색상을 선택해주세요"
                - data-testid: "phone-color-select"

핵심요구사항) 타입 정의를 추가할 것.
            - BasicInfoInput: { modelName: string; condition: string; color: string; manufacturer?: string; storage?: string; ram?: string; releaseYear?: number; }
            - src/components/phone-new/types.ts에 정의

핵심요구사항) useBasicInfo Hook을 구현할 것.
            1) 기능: 폼 필드 상태 관리 (아직 Zustand 연동 전 로컬 상태)
            2) 상태
                - modelName: string
                - condition: string
                - color: string
                - manufacturer?: string (자동 채움)
                - storage?: string (자동 채움)
                - ram?: string (자동 채움)
                - releaseYear?: number (자동 채움)
            3) 메서드
                - setModelName(value: string)
                - setCondition(value: string)
                - setColor(value: string)
                - loadModelDetails(modelName: string): Promise<void> (Supabase에서 모델 상세정보 조회)
                - reset(): void

핵심요구사항) Supabase 조회 로직
            1) 모델명 자동완성 데이터 출처
                - SELECT DISTINCT model_name FROM phones ORDER BY model_name
            2) 모델 선택 시 상세 정보 조회
                - SELECT manufacturer, storage, ram, release_year FROM phones WHERE model_name = ? LIMIT 1
