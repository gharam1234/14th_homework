아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - @01-common.mdc
            - @02-wireframe.mdc
            - @03-ui.mdc

조건-파일경로) 구현될 TSX 파일경로: src/components/phones-list/index.tsx
조건-파일경로) 구현될 CSS 파일경로: src/components/phones-list/styles.module.css

==============================================

핵심요구사항) 아이콘 필터 UI 영역을 구현할 것.

1) 아이콘 데이터 정의
   1-1) 카테고리 아이콘 배열을 다음 형식으로 정의할 것.
        - ['phone', 'tablet', 'laptop', 'watch']
   1-2) 이 배열을 상수로 관리할 것.
   1-3) data-testid="icon-filter" 속성을 추가할 것.

2) 가로 스크롤 UI 구현
   2-1) 아이콘들이 가로로 쭉 나열될 것.
   2-2) CSS overflow-x: scroll을 사용하여 옆으로 스크롤 가능하게 구현할 것.
   2-3) 스크롤바는 표시될 것.
   2-4) 각 아이콘은 일정한 간격(gap)을 유지할 것.

3) 선택 상태 UI
   3-1) 아이콘을 클릭하면 선택/선택 해제가 토글되도록 할 것.
   3-2) 선택된 아이콘에 테두리(border)를 표시할 것.
       - 테두리 색상: 프로젝트 색상 토큰 사용 (하드코딩 금지)
       - 테두리 두께: 2px 이상
   3-3) 미선택 아이콘은 테두리가 없을 것.
   3-4) 각 아이콘에 data-testid="icon-[카테고리명]" 속성을 추가할 것.
        - 예: data-testid="icon-phone", data-testid="icon-tablet"

4) 상태 관리 (단일 선택)
   4-1) 한 번에 한 개의 아이콘만 선택 가능할 것.
   4-2) 선택된 아이콘의 값을 상태로 관리할 것.
   4-3) 아이콘 클릭 시:
        - 이미 선택된 아이콘이면 선택 해제
        - 다른 아이콘이면 선택 전환
   4-4) 초기 상태는 어떤 아이콘도 선택되지 않은 상태일 것.

5) 로딩 및 에러 상태
   5-1) 데이터 로딩 중일 때 에러 메시지는 숨김 처리할 것.
   5-2) 에러 발생 시 다음 영역에 에러 메시지를 표시할 것.
        - data-testid="icon-filter-error"
        - 예: "필터를 불러올 수 없습니다."
   5-3) 에러 상태에서도 아이콘 필터는 표시될 것 (상호작용 가능).

핵심요구사항) 완성된 컴포넌트를 페이지에서 import하여 연결시킬 것.
                - 연결될 경로: src/app/phones/page.tsx
