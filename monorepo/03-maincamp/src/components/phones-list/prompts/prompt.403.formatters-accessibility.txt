핵심요구사항) 다국어·다통화 대응을 위한 추상화된 포맷터와 접근성 규칙을 구현할 것.

1) 포맷터 함수 구현 (src/commons/utils/formatters.ts)
   - formatPrice(price: number, currency: string = 'KRW'): string
     형식: "₩500,000" (KRW)
     규칙: 3자리마다 쉼표 추가, 통화 기호 포함

   - formatStorageCapacity(capacity: string): string
     형식: "256GB" → "256GB", "1TB" → "1TB"
     규칙: 단위 통일, 빈 값 처리 ('미제공')

   - formatDeviceCondition(condition: string): string
     매핑:
       - 'new' → '새상품'
       - 'like_new' → '미개봉'
       - 'excellent' → '상태좋음'
       - 'good' → '좋음'
       - 'fair' → '보통'
       - 'poor' → '나쁨'
     규칙: 빈 값 처리 ('미제공')

   - formatAddress(address: string, detail?: string): string
     형식: "서울시 강남구" (상세 주소 생략)
     규칙: 첫 번째 space까지만 표시, 빈 값 처리 ('위치 미표시')

   - formatSaleState(state: string): string
     매핑:
       - 'available' → '판매중'
       - 'reserved' → '예약중'
       - 'sold' → '판매완료'
     규칙: 빈 값 처리 ('상태 미표시')

   - formatTags(tags: string[] | null): string
     형식: "#tag1 #tag2" (공백으로 구분)
     규칙: 최대 3개까지만 표시, 초과 시 "+더보기" 추가

2) 커스텀 훅 구현 (src/components/phones-list/hooks/index.formatters.hook.ts)
   - usePhonePriceFormatter(price: number, currency: string): string
     → formatPrice(price, currency) 래퍼

   - usePhoneConditionFormatter(condition: string): string
     → formatDeviceCondition(condition) 래퍼

   - usePhoneAddressFormatter(address: string, detail?: string): string
     → formatAddress(address, detail) 래퍼

   - usePhoneSaleStateFormatter(state: string): string
     → formatSaleState(state) 래퍼

   - usePhoneTagsFormatter(tags: string[] | null): string
     → formatTags(tags) 래퍼

3) 접근성 규칙 전역 적용

   3-1) 이미지 접근성
        - 모든 <img> 태그에 alt 속성 추가 (의미있는 텍스트)
        - alt="[기기명] [모델명] 이미지" 형식
        - 예: alt="Apple iPhone 13 이미지"

   3-2) 포커스 접근성
        - 찜 버튼, 카드 링크에 tabindex 순서 확인
        - 찜 버튼: data-testid="favorite-button-{phoneId}"
        - 카드 링크: data-testid="phone-card-link-{phoneId}"

   3-3) 대체 텍스트
        - 아이콘 버튼에 aria-label 추가
        - 가격 표시: aria-label="가격: [포맷된 가격]"
        - 상태 표시: aria-label="판매 상태: [포맷된 상태]"

   3-4) 시맨틱 HTML
        - <article> 또는 <li>로 카드 감싸기
        - <button> 대신 semantic <a> 또는 <button> 사용
        - 제목: <h3> 사용

   3-5) 색상 의존성 제거
        - 판매 상태를 색상만으로 표현하지 말 것
        - 텍스트 라벨과 함께 표시 (예: "판매완료" 텍스트 + 회색 배경)

4) 페이징 제어 모듈화
   - usePhonePaginationControl 훅 생성
   - 기능:
     - canGoNext(): boolean (다음 페이지 가능 여부)
     - canGoPrevious(): boolean (이전 페이지 가능 여부)
     - getDisplayText(): string (현재 페이지 정보 표시, "1 / 5")
   - Zustand 상태 (totalCount, pageSize, currentPage) 활용

5) 무한 스크롤 대응 (선택)
   - useInfiniteScroll 훅 (선택 구현)
   - 기능: 스크롤 이벤트 감지, 마지막 아이템 감지 시 nextPage() 호출
   - data-testid="scroll-sentinel" 마크업 추가

6) 테스트
   - formatPrice(500000, 'KRW') === '₩500,000' 검증
   - formatStorageCapacity('256GB') === '256GB' 검증
   - formatDeviceCondition('like_new') === '미개봉' 검증
   - formatAddress('서울시 강남구 역삼동', '123') === '서울시 강남구' 검증
   - formatSaleState('reserved') === '예약중' 검증
   - formatTags(['tag1', 'tag2', 'tag3', 'tag4']) === '#tag1 #tag2 #tag3 +1더보기' 검증
   - 모든 이미지에 alt 속성이 있는지 검증
   - 찜 버튼과 링크에 aria-label이 있는지 검증
   - 페이지 네비게이션 텍스트가 올바르게 표시되는지 검증

7) 추가 요구사항
   - 포맷터 함수에서 null/undefined 안전 처리 (기본값 '미제공' 또는 '-')
   - 포맷터는 순수 함수(pure function)로 구현
   - 훅은 포맷터를 래핑하여 필요시 추가 로직(캐싱 등) 적용 가능하게 설계
   - 모든 포맷터 함수를 storybook story로 테스트하지 말 것
