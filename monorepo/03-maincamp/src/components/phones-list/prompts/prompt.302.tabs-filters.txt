핵심요구사항) 탭 기능을 다음과 같이 구현할 것.
                1) activeTab state 관리
                    - "selling" | "completed"
                    - 기본값: "selling"

                2) 탭 선택 로직
                    - 탭 클릭 시 activeTab 변경
                    - 해당 탭의 데이터만 표시

                3) 탭 UI 상태
                    - 활성 탭: 검은색 텍스트 + 하단 검은색 보더
                    - 비활성 탭: 회색 텍스트 + 보더 없음

핵심요구사항) 브랜드 필터를 다음과 같이 구현할 것.
                1) 필터 항목
                    - Apple, Samsung, Google, Xiaomi, Nothing, Sony, Motorola, LG, 기타
                    - 아이콘 + 텍스트 표시

                2) 필터 선택 로직
                    - 클릭 시 필터 활성화/비활성화
                    - selectedBrands state에 저장
                    - 활성 필터: 하이라이트 처리

                3) 필터 적용
                    - 선택된 브랜드의 카드만 표시
                    - 다중 선택 가능 (OR 조건)

핵심요구사항) 필터 상태를 localStorage에 저장할 것.
                1) 저장 데이터
                    - activeTab: "selling" | "completed"
                    - selectedBrands: string[]
                    - 키: "phones-filter"

                2) 로드 시
                    - localStorage에서 필터 상태 읽기
                    - 저장된 상태 복원

핵심요구사항) 필터 UI 상태를 다음과 같이 구현할 것.
                1) 활성 필터
                    - 아이콘: 색상 적용 (원본 색상)
                    - 텍스트: 검은색
                    - 배경: 연한 회색 (#f5f5f5)

                2) 비활성 필터
                    - 아이콘: 회색
                    - 텍스트: 회색
                    - 배경: 투명

핵심요구사항) Playwright 테스트를 TDD 기반으로 작성할 것.
                1) 테스트 파일경로: src/components/phones-list/tests/index.tabs.spec.ts
                2) 테스트 케이스

                    - 카드 필터링: 활성 탭의 카드만 표시
                    - 브랜드 필터 선택: 해당 브랜드 카드만 표시
                    - localStorage 저장: 새로고침 후 상태 복원 확인
