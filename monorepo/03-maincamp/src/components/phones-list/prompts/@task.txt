태스크2. 한정 특가 배너 영역

요구사항 1. 특가 데이터(배경, 메시지, CTA)를 모듈로 분리하고 노출 여부를 제어.
요구사항 2. “자급제 한정” 등 라벨을 다중 선택 가능 구조로 만들어 상태 값에 따라 주입.
요구사항 3. 배너 클릭 시 프로모션 상세/리스트로 라우팅하고 추적 이벤트를 기록.

태스크3. 중고폰 검색 필터 박스

요구사항 1. “즉시 구매 가능/예약 판매” 스위치(토글)의 상태를 한 곳에서 공통으로 관리하고, 다른 필터들이랑 같이 써도 문제없이 동작하게 만든다.

요구사항 2. 거래 가능한 날짜 범위를 달력(캘린더)에서 고를 수 있게 하고, 날짜를 선택하거나 초기화했을 때 각각 어떻게 동작할지(이벤트)를 구현한다.

요구사항 3. 모델명/키워드를 입력했을 때 “검색”·“판매글 등록” 버튼이 언제 활성화될지 기준을 정하고, 입력한 값이 올바른지 검사(유효성 체크)하는 기능을 만든다.

태스크4. 카테고리 아이콘 필터

요구사항 1. 아이콘 필터 목록을 데이터 구조로 정의하고 가로 스크롤 가능한 배열 상태로 관리.
요구사항 2. 단일/다중 선택 정책을 명시해 선택 토글 로직을 개발.
요구사항 3. 필터 변경 시 즉시 리스트 조회 API를 재호출하고 로딩·오류 상태를 처리.
태스크5. 중고폰 카드 그리드 리스트

요구사항 1. 서버 기기 데이터를 기반으로 페이징/무한 스크롤형 그리드 목록 로직 구현.
요구사항 2. 카드에 포함될 모델·용량·상태·해시태그·판매자·지역·가격·찜 상태를 표준 스키마로 구성.
요구사항 3. 카드 선택 시 상세 페이지 라우팅과 인터랙션(상태 강조 등)을 제어.
태스크6. 공통 인터랙션 및 상태 관리

요구사항 1. 찜 버튼 동작에 로그인 검증, API 호출, 낙관적 업데이트·실패 롤백을 구현.
요구사항 2. 리스트 요청 로딩·에러·빈 상태를 통합 처리하고 페이징/무한 스크롤 제어를 모듈화.
요구사항 3. 다국어·다통화 대응을 위해 텍스트/가격 포맷터를 추상화하고 포커스·대체 텍스트 등 접근성 규칙을 전역 적용.
