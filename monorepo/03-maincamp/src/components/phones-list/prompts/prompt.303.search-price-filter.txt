### 상태 관리 가이드
            - 검색 및 가격 상태는 `src/components/phones-list/stores/use-phone-list.store.ts`에서 관리한다. 파일이 없다면 이번 작업에서 생성하고 `searchKeyword`, `minPrice`, `maxPrice`, `setSearchKeyword`, `setPriceRange` API를 제공한다.
            - 기존 `samplePhones` 의 `price` 값이 문자열이라면 훅 내부에서 숫자로 변환(`Number(price.replace(/,/g, ""))`)하여 비교하도록 명시한다.

==============================================

핵심요구사항) 검색 기능을 다음과 같이 구현할 것.
                1) 검색 입력 필드
                    - placeholder: "모델명이나 기기명을 검색해 주세요."
                    - 실시간 필터링 (입력 시 즉시 반영)

                2) 검색 로직
                    - title, description에서 검색어 매칭
                    - 대소문자 구분 없음
                    - 부분 검색 지원

                3) 검색 버튼
                    - 클릭 시 검색 수행
                    - 또는 Enter 키 누르기

핵심요구사항) 가격 범위 필터를 다음과 같이 구현할 것.
                1) 가격 범위 입력
                    - 최소 가격 입력 필드
                    - 최대 가격 입력 필드
                    - 또는 달력/범위 선택 UI (Figma 디자인 기준)

                2) 가격 필터 로직
                    - minPrice ≤ card.price ≤ maxPrice 조건
                    - 입력 시 즉시 반영

                3) 가격 포맷
                    - 숫자만 입력 가능
                    - 쉼표 자동 추가 (예: 1,000,000)

핵심요구사항) 필터 상태를 localStorage에 저장할 것.
                1) 저장 데이터
                    - searchKeyword: string
                    - minPrice: number
                    - maxPrice: number
                    - 키: "phones-search-filter"

                2) 로드 시
                    - localStorage에서 필터 상태 읽기
                    - 저장된 검색어와 가격 범위 복원

핵심요구사항) 필터 초기화 기능을 구현할 것.
                1) 초기화 버튼
                    - 클릭 시 모든 필터 제거
                    - searchKeyword: ""
                    - minPrice: 0
                    - maxPrice: null

                2) UI 표시
                    - 필터가 활성화되었을 때만 "초기화" 버튼 표시

핵심요구사항) 검색/필터 결과를 다음과 같이 표시할 것.
                1) 결과 없을 때
                    - "검색 결과가 없습니다." 메시지 표시
                    - 카드 그리드 숨기기

                2) 결과 있을 때
                    - 필터링된 카드 표시
                    - "결과 N개" 카운트 표시 (선택)

핵심요구사항) Playwright 테스트를 TDD 기반으로 작성할 것.
                1) 테스트 파일경로: src/components/phones-list/tests/index.search.spec.ts
                2) 테스트 케이스
                    - 검색어 입력: 해당 검색어 포함 카드만 표시
                    - 가격 범위 설정: 범위 내 카드만 표시
                    - 검색 + 가격 필터: 두 조건 모두 적용
                    - 초기화 버튼: 모든 필터 제거
                    - localStorage 저장: 새로고침 후 상태 복원
